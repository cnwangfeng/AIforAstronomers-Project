\chapter{可视化与科研图表}

\section{本章导读：图表是科学论证，不是装饰}

很多学生第一次学习绘图时，会把目标理解成“用 matplotlib 把数据画出来”。这只是最表层的技能。科研图表真正承担的是论证任务：它把数据关系、测量不确定度、模型诊断和解释边界组织成读者可以检查的视觉证据。

一张图如果只有点、线和颜色，但没有变量、单位、尺度、样本选择、误差和读图说明，那么它很可能只是“漂亮输出”，还不是科学图表。反过来，一张图即使很朴素，只要能清楚回答“比较什么、证据在哪里、不能过度解释什么”，它就已经开始发挥科研作用。

本章承接前面几章：

\begin{itemize}
    \item Linux 章让你知道图像文件应该保存在哪里；
    \item Git 章让你知道图表来自哪个版本；
    \item 脚本章让你知道图表如何从稳定入口生成；
    \item I/O 章让你知道图表输入是否经过检查；
    \item 本章则讨论图表本身如何成为证据。
\end{itemize}

\section{学习目标与训练目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 解释为什么科研图表不是分析结束后的装饰；
    \item 区分变量、单位、对象、尺度和视觉编码在图表中的作用；
    \item 根据关系类型选择散点图、折线图、直方图、误差棒图、图像图或残差图；
    \item 理解线性坐标、对数坐标和归一化显示的适用场景；
    \item 用 claim--evidence--limit 框架把图表转化为科学表述；
    \item 识别颜色、点大小、坐标范围、bin 宽度和图例可能造成的误导；
    \item 保存图像文件时记录数据来源、代码版本、参数和输出路径。
\end{itemize}

训练目标是完成一个小型图表交付：从一份教学数据中生成主图和一个诊断图，写出图注、读图说明、自审清单和 figure record。

\section{从数据到图像：绘图前已经有假设}

图表不是中性地把数据“显示出来”。在画图前，你已经做出一系列选择：

\begin{enumerate}
    \item 选择哪些样本进入图；
    \item 选择哪些变量作为横轴和纵轴；
    \item 选择是否做单位转换或归一化；
    \item 选择线性坐标还是对数坐标；
    \item 选择用点、线、颜色、大小、形状还是误差棒编码信息；
    \item 选择是否标注对象名称或理论参考线。
\end{enumerate}

这些选择共同决定读者会看到什么，也决定读者容易忽略什么。因此，绘图不是中立动作，而是一次视觉建模。

\begin{defbox}[科学图表]
科学图表是把数据、关系、诊断和解释边界组织成视觉证据的表达方式。它的目标不是单纯美观，而是帮助读者检验一个科学判断。
\end{defbox}

\section{准备变量：图表从字段选择开始}

以行星轨道数据为例，最小准备过程是：

\begin{examplebox}[读取教学数据并准备变量]
\begin{lstlisting}[style=pythonstyle]
import csv
from pathlib import Path

data_path = Path("data/planetary_orbits_demo.csv")
with data_path.open(newline="", encoding="utf-8") as handle:
    rows = list(csv.DictReader(handle))

planet_names = [row["planet"] for row in rows]
semi_major_axis_au = [float(row["semi_major_axis_au"]) for row in rows]
orbital_period_years = [float(row["orbital_period_years"]) for row in rows]
\end{lstlisting}
\end{examplebox}

这段代码已经完成三件科学选择：

\begin{itemize}
    \item 选择对象集合：太阳系行星教学样本；
    \item 选择横轴变量：半长轴；
    \item 选择纵轴变量：轨道周期。
\end{itemize}

如果字段含义、单位或样本选择没有写清，后面的图再漂亮也不可靠。

\section{变量、单位与对象名称}

图表的轴标签不是形式要求，而是读者理解图像的入口。对于行星轨道示例，横轴和纵轴可以写成：

\[
a\;[\mathrm{au}], \qquad P\;[\mathrm{yr}].
\]

其中 \(a\) 是半长轴，\(P\) 是轨道周期。单位 \(\mathrm{au}\) 和 \(\mathrm{yr}\) 让读者知道数值尺度。对于小样本且对象身份重要的图，还应该考虑标注对象名称。

\begin{examplebox}[最小散点图]
\begin{lstlisting}[style=pythonstyle]
import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(6, 4))
ax.scatter(semi_major_axis_au, orbital_period_years, color="tab:blue")

for name, x_value, y_value in zip(
    planet_names, semi_major_axis_au, orbital_period_years
):
    ax.annotate(name, (x_value, y_value), xytext=(4, 4),
                textcoords="offset points", fontsize=8)

ax.set_xlabel("Semi-major axis [au]")
ax.set_ylabel("Orbital period [yr]")
ax.set_title("Orbital period vs. semi-major axis")
ax.grid(alpha=0.3)
\end{lstlisting}
\end{examplebox}

这段代码中的每一行都对应一个设计选择：点表示离散对象，标注表示对象身份，轴标签说明变量和单位，网格帮助读数但不应抢走注意力。

\section{为什么推荐 \texttt{fig, ax} 写法}

Matplotlib 有多种写法。入门时常见的 \texttt{plt.plot(...)} 很方便，但当图表变复杂时，推荐使用：

\begin{lstlisting}[style=pythonstyle]
fig, ax = plt.subplots()
\end{lstlisting}

这里 \texttt{fig} 表示整张画布，\texttt{ax} 表示其中一个坐标轴区域。这样写的好处是：

\begin{itemize}
    \item 图中有多个子图时，每个 \texttt{ax} 的职责清楚；
    \item 保存图像、添加颜色条、调整坐标轴更容易控制；
    \item 函数可以接收 \texttt{ax} 参数，从而把绘图逻辑复用到不同图里。
\end{itemize}

科研图表经常需要主图、残差图、颜色条和多个面板。越早理解 \texttt{fig} 与 \texttt{ax} 的分工，后面越不容易陷入“能画出来但无法维护”的状态。

\section{对数坐标：当关系跨越数量级}

许多天文和物理量跨越多个数量级。线性坐标会把小值挤在一起，导致结构难以观察。对数坐标适合展示乘法关系、幂律关系和大动态范围数据。

行星轨道示例背后是开普勒第三定律：

\[
P^2 \propto a^3.
\]

取对数可得：

\[
\log P = \frac{3}{2}\log a + C.
\]

这说明如果原始关系是幂律，在对数坐标下会更接近直线。

\begin{examplebox}[使用对数坐标]
\begin{lstlisting}[style=pythonstyle]
ax.set_xscale("log")
ax.set_yscale("log")
\end{lstlisting}
\end{examplebox}

对数坐标不是为了显得高级，而是为了让适合的结构可见。需要注意的是，对数轴不能直接显示非正数；如果数据包含零或负值，必须先理解这些值的来源。

\section{在图上画参考关系}

如果图表要支持一个物理关系，仅画散点往往不够。以开普勒第三定律为例，可以在图中画一条参考关系：

\[
P = a^{3/2},
\]

其中 \(a\) 用 au、\(P\) 用 year。教学代码可以写成：

\begin{examplebox}[参考关系线]
\begin{lstlisting}[style=pythonstyle]
axis_grid = [0.4, 0.7, 1.0, 2.0, 5.0, 10.0, 30.0]
reference_period = [axis ** 1.5 for axis in axis_grid]

ax.plot(axis_grid, reference_period,
        color="black", linestyle="--", label="P = a^(3/2)")
ax.legend()
\end{lstlisting}
\end{examplebox}

参考线的作用不是替代数据，而是让读者知道你在比较什么理论或经验关系。没有参考关系的散点图只能说“看起来相关”；有了参考关系，图表才更接近“数据是否支持某个预期关系”的科学论证。

\section{从图到论断：claim--evidence--limit}

科研图表必须能转化成文字判断。最小框架是：

\begin{enumerate}
    \item \textbf{Claim}：这张图支持什么论断；
    \item \textbf{Evidence}：图中的哪些结构支持该论断；
    \item \textbf{Limit}：这个论断在哪些条件下成立，哪里不能过度解释。
\end{enumerate}

\begin{examplebox}[行星轨道图的三句式说明]
Claim：轨道周期随半长轴增大而增大，并且更接近幂律关系。

Evidence：在对数坐标下，太阳系行星样本呈现近似直线趋势，符合 \(P^2 \propto a^3\) 的直觉。

Limit：样本只包含教学用太阳系行星数据，不能直接推广到所有系外行星系统。
\end{examplebox}

这个三句式很朴素，但非常重要。它防止图表变成装饰，也防止读者从图中过度读出没有被数据支持的结论。

\section{误差棒：把测量不确定度画出来}

如果每个点都有测量不确定度，就应该考虑误差棒。误差棒不是为了让图更复杂，而是告诉读者点的位置有多可靠。

\begin{examplebox}[最小误差棒图]
\begin{lstlisting}[style=pythonstyle]
fig, ax = plt.subplots(figsize=(6, 4))
ax.errorbar(time_days, magnitude, yerr=mag_error,
            fmt="o", capsize=3, color="tab:blue")
ax.set_xlabel("Time [day]")
ax.set_ylabel("Magnitude [mag]")
\end{lstlisting}
\end{examplebox}

误差棒需要解释：

\begin{itemize}
    \item 它表示 1 sigma、标准误、置信区间，还是其他量；
    \item 是否只包含随机误差；
    \item 是否存在系统误差没有画出来；
    \item 误差是否与数据点相互独立。
\end{itemize}

没有说明的误差棒，很容易制造“看起来很严谨”的错觉。

\section{残差图：让模型接受检查}

如果图中包含模型线，仅仅展示模型和数据重合是不够的。残差图常常比主图更诚实。

设观测值为 \(y_i\)，模型预测为 \(\hat{y}_i\)，残差为：

\[
r_i = y_i - \hat{y}_i.
\]

如果模型合适，残差通常应围绕零随机分布；如果残差随 \(x\) 呈现结构，说明模型遗漏了某种关系。

\begin{examplebox}[最小残差计算]
\begin{lstlisting}[style=pythonstyle]
residuals = [observed - predicted
             for observed, predicted in zip(y_values, y_pred)]

fig, ax = plt.subplots(figsize=(6, 3))
ax.axhline(0, color="black", linewidth=1)
ax.scatter(x_values, residuals, color="tab:orange")
ax.set_xlabel("x")
ax.set_ylabel("Residual")
\end{lstlisting}
\end{examplebox}

后续机器学习评估、回归、分类和案例章节都会反复使用这种诊断思想。

\section{视觉编码：颜色、大小和范围会影响判断}

视觉编码不是装饰。颜色、点大小、透明度、坐标范围、bin 宽度、图例顺序都会影响读者判断。

\begin{warnbox}[视觉编码中的常见误导]
\begin{itemize}
    \item 用彩虹色图让连续量产生虚假分界；
    \item 坐标轴截断让差异看起来更大；
    \item 点太大或不透明，掩盖密集区域；
    \item 直方图 bin 宽度改变结论观感；
    \item 图例颜色和正文类别解释不一致；
    \item 没有说明筛选条件，使图中样本看似完整。
\end{itemize}
\end{warnbox}

图表自审时，不要只问“好不好看”，要问“有没有诱导读者相信数据并不支持的东西”。

\section{颜色选择与可读性}

颜色最容易被误当成审美问题。实际上，颜色首先是信息编码。一个更稳妥的原则是：

\begin{itemize}
    \item 连续量使用连续色图，例如灰度、viridis 类顺序色图；
    \item 离散类别使用彼此区分清楚的分类颜色；
    \item 不要只靠红绿区分关键类别；
    \item 打印成灰度后仍应尽量能读；
    \item 颜色条或图例必须说明颜色含义。
\end{itemize}

如果颜色承载了科学信息，就必须像轴标签一样解释。否则读者可能把颜色差异误读成数据差异，或完全看不出你想表达的结构。

\section{图像图与颜色条}

天文图像、探测器图像和二维模拟结果常常用图像图展示。此时颜色表示数值大小，而不是类别。颜色条就是图像图的单位说明。

\begin{examplebox}[最小图像图]
\begin{lstlisting}[style=pythonstyle]
fig, ax = plt.subplots(figsize=(5, 4))
image_handle = ax.imshow(image_data, origin="lower", cmap="gray")
fig.colorbar(image_handle, ax=ax, label="Counts")
ax.set_xlabel("x [pixel]")
ax.set_ylabel("y [pixel]")
\end{lstlisting}
\end{examplebox}

图像显示还需要考虑：

\begin{itemize}
    \item \texttt{origin} 是否符合图像坐标习惯；
    \item 灰度范围是否被截断；
    \item 颜色条单位是什么；
    \item 是否需要对数、平方根或百分位拉伸；
    \item 图像是否已经背景修正。
\end{itemize}

\section{保存图像：输出文件也是科研记录}

图像保存不只是最后一行代码。你需要记录图像来自哪份数据、哪段代码、哪个 commit、什么参数和什么输出格式。

\begin{examplebox}[保存图像]
\begin{lstlisting}[style=pythonstyle]
fig.savefig("figures/orbit_period_vs_axis.png",
            dpi=200, bbox_inches="tight")
\end{lstlisting}
\end{examplebox}

课程项目中，一张最终图至少应能追踪到：

\begin{itemize}
    \item 输入数据文件；
    \item 绘图脚本或 notebook；
    \item 代码版本；
    \item 图像文件路径；
    \item 坐标尺度和关键参数；
    \item 图注和解释限制。
\end{itemize}

\section{把绘图流程写成脚本}

最终图不应该只存在于 notebook 的某个执行状态里。一个最小绘图脚本可以长这样：

\begin{examplebox}[绘图脚本骨架]
\begin{lstlisting}[style=pythonstyle]
import csv
from pathlib import Path

import matplotlib.pyplot as plt


def read_rows(data_path):
    with data_path.open(newline="", encoding="utf-8") as handle:
        return list(csv.DictReader(handle))


def prepare_variables(rows):
    names = [row["planet"] for row in rows]
    axis_au = [float(row["semi_major_axis_au"]) for row in rows]
    period_year = [float(row["orbital_period_years"]) for row in rows]
    return names, axis_au, period_year


def make_figure(data_path, figure_path):
    rows = read_rows(data_path)
    names, axis_au, period_year = prepare_variables(rows)

    fig, ax = plt.subplots(figsize=(6, 4))
    ax.scatter(axis_au, period_year, color="tab:blue")
    ax.set_xscale("log")
    ax.set_yscale("log")

    for name, x_value, y_value in zip(names, axis_au, period_year):
        ax.annotate(name, (x_value, y_value), xytext=(4, 4),
                    textcoords="offset points", fontsize=8)

    ax.set_xlabel("Semi-major axis [au]")
    ax.set_ylabel("Orbital period [yr]")
    ax.set_title("Orbital period vs. semi-major axis")
    ax.grid(alpha=0.3)

    figure_path.parent.mkdir(exist_ok=True)
    fig.savefig(figure_path, dpi=200, bbox_inches="tight")


def main():
    make_figure(
        Path("data/planetary_orbits_demo.csv"),
        Path("figures/orbit_period_vs_axis.png"),
    )


if __name__ == "__main__":
    main()
\end{lstlisting}
\end{examplebox}

这个脚本仍然很小，但已经包含完整结构：读取数据、准备变量、绘图、保存图像和命令行入口。后续报告应能指向这个脚本，而不是只说“图是 notebook 画的”。如果要进一步提高可复现性，可以把输入路径、输出路径和坐标尺度改成命令行参数，并把 commit hash 写入 figure record。

\end{lstlisting}
\end{examplebox}

如果你不能填写这份模板，说明图表还没有准备好进入报告。

\section{图表交付前的自审清单}

\begin{enumerate}
    \item 横轴和纵轴是否有变量名和单位；
    \item 图表类型是否服务于问题；
    \item 坐标尺度是否合理；
    \item 颜色、大小、透明度和图例是否解释清楚；
    \item 是否需要误差棒或残差图；
    \item 是否说明样本筛选条件；
    \item 图像文件是否可从脚本重新生成；
    \item 图注是否包含 claim、evidence 和 limit；
    \item 输出文件是否记录在 Git 或结果记录中。
\end{enumerate}

这份清单会在后续 HR 图、FITS 图像、光谱、时间序列、模型诊断图和 capstone 报告中反复使用。

\section{配套 notebook 如何训练}

本章配套 notebook 会完成一条小型图表工作流：

\begin{enumerate}
    \item 读入教学数据；
    \item 检查字段和单位；
    \item 绘制最小散点图；
    \item 尝试线性坐标和对数坐标；
    \item 添加对象标注；
    \item 生成一个诊断或辅助图；
    \item 保存图像文件；
    \item 写出图注和 figure record。
\end{enumerate}

notebook 的作用是训练你把图读出来，而不是只把图画出来。

\section{常见错误与诊断}

\begin{warnbox}[科研图表中的典型问题]
\begin{itemize}
    \item 轴标签只有变量名，没有单位；
    \item 图表类型和问题不匹配；
    \item 对数坐标使用后没有说明；
    \item 颜色编码没有图例或颜色条；
    \item 坐标范围截断导致差异被夸大；
    \item 只展示主图，不做误差或残差诊断；
    \item 图注只描述“画了什么”，没有说明支持什么论断；
    \item 图像文件无法从脚本或 notebook 重新生成。
\end{itemize}
\end{warnbox}

诊断图表问题时，可以从三个问题开始：这张图比较什么？图中的哪个结构支持我的结论？这张图不能说明什么？

\section{AI 助手如何帮助你}

AI 助手可以帮助你改进图注、检查轴标签、建议图表类型、指出可能误导的视觉编码、审查 claim--evidence--limit 是否匹配。但它不能替你确认数据是否可靠，也不能替你决定科学结论。

一个合适的提示词是：

\begin{lstlisting}[style=terminal]
请审查这张科研图的说明。重点检查变量和单位是否清楚，
图表类型是否匹配问题，claim 是否被图中结构支持，
limit 是否写得足够具体。
\end{lstlisting}

如果让 AI 生成绘图代码，也应该要求它解释每个视觉编码选择，而不是只给出可运行代码。

\section{练习与交付}

\begin{exercisebox}[基础练习]
读取一份包含两个连续变量的小型 CSV，绘制散点图。要求横轴、纵轴、标题和单位完整。
\end{exercisebox}

\begin{exercisebox}[尺度练习]
对同一组数据分别使用线性坐标和对数坐标。比较哪一种更能展示结构，并说明原因。
\end{exercisebox}

\begin{exercisebox}[诊断练习]
给定一组观测值和模型预测值，计算残差并画残差图。判断残差是否呈现系统结构。
\end{exercisebox}

\begin{exercisebox}[交付练习]
保存最终图像，写一份 figure record，至少包含数据来源、绘图入口、变量单位、坐标尺度、claim、evidence、limit 和 commit hash。
\end{exercisebox}

\section{本章小结}

科研图表的目标不是“画得好看”，而是让数据关系变成可检查的视觉证据。好的图表必须说明变量、单位、尺度、样本、视觉编码和解释边界；好的图注必须说明 claim、evidence 和 limit；好的交付还必须让图像可以从数据和脚本重新生成。掌握这套习惯后，后面的 HR 图、光谱图、时间序列图、模型评估图和 capstone 报告都会更可靠。
