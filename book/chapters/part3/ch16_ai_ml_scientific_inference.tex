\chapter{什么是 AI、机器学习与科学推断}

\section{学习目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 区分规则方法、统计建模、机器学习和科学推断这几个常被混用但并不相同的概念；
    \item 理解一个最小监督学习问题如何写成“样本、特征、标签、模型与损失函数”的形式；
    \item 说明为什么在科学任务中，预测能力、可解释性和物理机制并不是同一件事；
    \item 理解 baseline 在科研机器学习中的作用，不把“复杂模型”误当成“更科学”；
    \item 从一个极小恒星分类示例出发，看清规则方法和数据驱动方法之间的联系与差别。
\end{itemize}

\section{为什么在 Part II 之后进入机器学习}

在前面的 Part II 里，我们已经处理了星表、图像、光谱、时间序列和实验数据。那一部分训练的重点，是把科学数据看成有单位、有误差、有采样条件和物理背景的对象。只有到了这个基础上，机器学习才真正值得引入。

原因很简单：机器学习并不会自动替你理解数据。如果你不知道一列量的单位、不知道哪些样本有严重选择效应、不知道观测误差会怎样扭曲边界，那么再复杂的模型也只是在学习这些混乱本身。

因此，本书对机器学习的立场非常明确：

\begin{itemize}
    \item 它不是“替代科学推理”的捷径；
    \item 它不是“只要分数更高就一定更好”的竞赛工具；
    \item 它是帮助我们组织证据、建立映射、量化不确定性和比较假设的一组方法。
\end{itemize}

\section{AI、统计模型与机器学习：边界在哪里}

今天“AI”这个词经常被用来指代非常不同的对象。为了避免概念混乱，本章先做一个克制但清楚的区分。

\subsection{规则方法}

规则方法是由研究者直接写出判断逻辑。例如：

\begin{itemize}
    \item 若绝对星等很暗，则优先判为白矮星；
    \item 若颜色很红且本征亮度高，则优先判为巨星；
    \item 若某段频率范围内功率高于阈值，则标记为周期候选。
\end{itemize}

规则方法的优点是可解释、直接、便于调试；缺点是边界可能过硬，难以适应复杂数据分布。

\subsection{统计模型}

统计模型通常从一组参数化关系出发，例如线性回归、泊松模型或高斯误差模型。它们强调：

\begin{itemize}
    \item 数据由什么分布产生；
    \item 参数具有什么含义；
    \item 误差如何进入推断。
\end{itemize}

在科学中，统计模型常常比“黑箱预测”更接近推断传统，因为它更容易和误差模型、似然函数、先验知识结合。

\subsection{机器学习模型}

机器学习模型则更强调：\textbf{从数据中学习一个映射关系或决策边界。}  
它往往不要求研究者先写出完整的机制方程，而是通过训练样本去学习

\[
\hat{y} = f_\theta(x),
\]

其中 $x$ 是输入特征，$\theta$ 是模型参数，$\hat{y}$ 是输出预测。

\begin{defbox}[本书中的机器学习]
在本书语境里，机器学习主要指利用数据学习输入到输出映射的一类方法。它可以很简单，比如最近邻、线性回归；也可以很复杂，比如神经网络和基础模型。
\end{defbox}

\section{科学推断不是简单等于“预测”}

这是本科生进入机器学习时最需要建立的一条边界。

一个模型在测试集上预测得很准，说明它学会了某种稳定的映射；但这并不自动意味着：

\begin{itemize}
    \item 它抓住了真实的物理机制；
    \item 它的内部表示与自然规律一一对应；
    \item 它可以安全外推到新仪器、新波段或新样本范围；
    \item 它的高分一定比一个更简单、更透明的模型更有科学价值。
\end{itemize}

例如，一个光度红移模型也许能很好地从颜色预测红移，但这不等于它已经解释了星系形成与演化；一个形态分类器也许能很好地区分旋涡星系和椭圆星系，但这不等于它知道这些形态差异的动力学成因。

\begin{protip}[预测能力与机制解释要分开讨论]
在科学工作里，一个模型可以在“预测上有用”但在“机制上有限”；也可以在“预测一般”但在“参数可解释性上很强”。不要让单一分数替代这两类判断。
\end{protip}

\section{一个最小监督学习问题怎么写}

很多后续章节都会使用监督学习，所以本章先把它写成最小数学形式。设我们有一个带标签的数据集：

\[
\mathcal{D} = \{(x_i, y_i)\}_{i=1}^{N},
\]

其中：

\begin{itemize}
    \item $x_i$ 是第 $i$ 个样本的特征向量；
    \item $y_i$ 是对应标签或目标值；
    \item $N$ 是样本数。
\end{itemize}

训练一个模型，意味着寻找参数 $\theta$，使得模型输出 $f_\theta(x_i)$ 与真实目标 $y_i$ 尽可能一致。通常会引入一个损失函数 $L$，并最小化经验风险：

\[
\hat{R}(\theta) = \frac{1}{N}\sum_{i=1}^{N} L\bigl(f_\theta(x_i), y_i\bigr).
\]

这条式子本身非常重要，因为它告诉我们：

\begin{itemize}
    \item 模型训练从来不是抽象魔法，而是在最小化某个明确定义的误差准则；
    \item 你选择什么损失函数，就在强调什么类型的错误；
    \item 训练结果总是与数据集、特征定义和标签质量绑定在一起。
\end{itemize}

\section{特征、标签与目标：三者不能混为一谈}

初学者经常把“表中的每一列”都视为可以直接喂给模型的内容，但在科学任务里，特征工程本身就包含理论判断。

\begin{itemize}
    \item \textbf{特征} 是模型可以读取的输入，例如颜色、绝对星等、谱线深度、统计量；
    \item \textbf{标签} 是希望模型学习预测的对象，例如恒星类别、星系类型、红移区间；
    \item \textbf{目标} 则是你真正想解决的科学问题。
\end{itemize}

三者并不总是一一对应。比如你真正关心的是“恒星演化阶段”，但在教学数据里只能先用人工定义的 \texttt{stellar\_class} 当作代理标签；你真正关心的是“物理参数”，但暂时只能先做一个二分类初筛。

这也是为什么很多科学机器学习项目，第一步不是“选模型”，而是“问清楚标签到底代表什么”。

\section{为什么 baseline 是机器学习里的认识论锚点}

一旦出现复杂模型，人很容易产生一种错觉：只要模型更复杂、参数更多、名称更前沿，就一定比简单方法更可靠。科学训练必须主动抵抗这种错觉。

baseline 的作用就在这里。它像一根锚，把问题钉回到最基本的比较上：

\begin{itemize}
    \item 如果一个手工规则已经能达到不错效果，复杂模型到底多带来了什么；
    \item 如果一个简单线性模型已经解释主要趋势，非线性模型的额外提升是否稳定；
    \item 如果复杂模型无法明显优于 baseline，问题是否根本不值得引入它。
\end{itemize}

\begin{examplebox}[恒星分类规则 baseline]
\begin{lstlisting}[style=pythonstyle]
def classify_star_rule(bp_rp, absolute_g_mag):
    if absolute_g_mag > 10.0:
        return "white_dwarf"
    if absolute_g_mag < 2.0 and bp_rp > 1.2:
        return "giant"
    return "main_sequence"
\end{lstlisting}
\end{examplebox}

这个例子并不复杂，但教学意义很强。它迫使我们先问：\textbf{颜色和绝对星等是否已经携带了足够多的物理结构？} 如果答案是“是”，那么任何更复杂的方法都必须在这个前提上提供额外价值。

\section{一个极小恒星分类例子：从物理量到学习问题}

本章配套 notebook 使用了一个小型 Gaia 风格教学样本。每个样本最终都会被组织成：

\begin{itemize}
    \item 颜色指数 \texttt{bp\_rp}；
    \item 由表观星等和视差计算出的绝对星等 \texttt{absolute\_g\_mag}；
    \item 教学标签 \texttt{stellar\_class}。
\end{itemize}

一旦写成这个形式，我们实际上已经拥有了一个最小监督学习任务：

\[
x_i = (\texttt{bp\_rp}, \texttt{absolute\_g\_mag}), \qquad
y_i \in \{\texttt{main\_sequence}, \texttt{giant}, \texttt{white\_dwarf}\}.
\]

值得注意的是，这里之所以适合作为教学起点，是因为输入特征本身并非任意数值，而是已经有明确物理直觉：

\begin{itemize}
    \item 颜色近似反映温度；
    \item 绝对星等近似反映本征亮度；
    \item 不同恒星类群在 HR 图上确实存在结构差异。
\end{itemize}

也就是说，这个学习任务不是凭空制造出来的，而是直接长在恒星物理结构之上。

\section{规则方法与数据驱动方法的联系}

本章 notebook 不会一上来就进入完整训练/测试流程，而是先做一件更基础的事：比较一个规则 baseline 与一个极简的数据驱动方法。

数据驱动方法的最小直觉可以是：对每一类样本，先计算它们在特征空间中的中心；再把一个新样本分到最近的类别中心。这类方法不需要你手写复杂阈值，而是让数据自己决定“每一类大致在什么位置”。

规则方法和数据驱动方法的差别，不是“前者落后、后者先进”，而是：

\begin{itemize}
    \item 规则方法把边界直接写出来；
    \item 数据驱动方法让边界由样本分布间接形成；
    \item 二者都依赖特征定义，也都离不开科学判断。
\end{itemize}

\section{为什么评估设计在下一章还要单独展开}

到了这里，读者很自然会问：既然已经有模型，为什么这一章不直接展开完整评估？原因是本章的重点还在概念边界，而不是流程细节。

真正严肃的模型比较，需要进一步回答：

\begin{itemize}
    \item 训练集和测试集如何划分；
    \item 小样本场景中如何避免某一类完全缺失；
    \item 分数到底是训练分数还是测试分数；
    \item 一个高分是否只是巧合于某次切分。
\end{itemize}

这些内容会在下一章系统展开。本章要先把地基打稳：\textbf{什么是学习问题、为什么需要 baseline、为什么预测与解释不能混淆。}

\section{机器学习在科学里最常见的四类任务}

为了给后续章节建立地图，本书接下来会反复回到四类基本任务：

\begin{itemize}
    \item \textbf{分类}：输出离散标签；
    \item \textbf{回归}：输出连续量；
    \item \textbf{无监督学习}：在没有显式标签时寻找结构；
    \item \textbf{异常检测}：从大量样本中寻找稀有或不典型对象。
\end{itemize}

无论是哪一类，最根本的问题都没有变：特征从哪里来、标签是否可信、评估是否合理、结果能否回到科学判断。

\section{选择效应、样本偏差与领域迁移}

科学数据很少满足“训练和测试来自完全相同分布”这种理想条件。现实中更常见的是：

\begin{itemize}
    \item 训练集来自亮源，而你想预测更暗的样本；
    \item 训练集来自某个巡天，而你想外推到另一台仪器；
    \item 标签来自人工筛选，本身就带着先验偏向；
    \item 某些稀有类别在训练集中样本极少，模型天然学不到。
\end{itemize}

这意味着机器学习结果必须放回观测条件和数据来源里解释。一个模型也许并不是“失效”，它可能只是被用到了超出样本支持范围的地方。

\begin{warnbox}[进入机器学习前最常见的概念错误]
\begin{itemize}
    \item 把“预测得准”直接等同于“理解了物理”；
    \item 在没有 baseline 的情况下直接引入复杂模型；
    \item 忘记标签本身也可能带着误差和偏见；
    \item 把表中所有列都当成理所当然的输入特征；
    \item 完全不问训练样本和目标样本是否来自相同观测条件。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这一章，AI 助手适合帮助你：

\begin{itemize}
    \item 帮你把某个监督学习问题重新写成“特征、标签、损失函数”的结构；
    \item 检查一个 baseline 是否足够清楚、是否真的能作为比较起点；
    \item 帮你解释规则方法、统计模型和机器学习方法各自的适用边界；
    \item 协助你把“预测效果”与“物理解释”分成两套不混淆的论证。
\end{itemize}

但某个标签是否可信、某个模型提升是否值得、以及一个结果是否真的推动了科学理解，仍然需要你自己做判断。

\section{本章小结}

本章的核心，不是让你立刻掌握某个具体模型，而是先建立机器学习进入科学问题时最重要的概念边界：学习问题如何形式化、baseline 为什么不可省、预测和推断为什么必须区分、以及模型结果为什么必须放回数据来源和物理背景中解释。只要这些边界真正建立起来，后面的分类、回归、无监督学习和异常检测章节就不会沦为纯粹的“调包库函数”练习。
