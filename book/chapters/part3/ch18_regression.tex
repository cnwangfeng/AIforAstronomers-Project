\chapter{回归}

\section{学习目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 区分回归任务与分类任务，并理解为什么很多科学问题天然是连续量预测；
    \item 从最小线性模型出发，写出回归问题的基本形式和最小二乘目标；
    \item 理解 MAE、RMSE 和残差各自刻画了什么，不再只盯着一个数字；
    \item 说明为什么多项式特征可以提升表达能力，同时也可能带来过拟合与外推风险；
    \item 把一个教学版光度红移问题走通为“切分 $\rightarrow$ baseline $\rightarrow$ 训练 $\rightarrow$ 残差分析 $\rightarrow$ 物理解读”的完整流程。
\end{itemize}

\section{为什么回归在科学里无处不在}

在分类任务中，我们问的是“它属于哪一类”；在回归任务中，我们问的是“这个量具体是多少”。后者在天文和物理里非常常见，例如：

\begin{itemize}
    \item 根据测光颜色估计星系红移；
    \item 根据光谱特征估计恒星温度、金属丰度或表面重力；
    \item 根据实验数据估计衰减时间常数、质量或能量参数；
    \item 根据多波段观测预测某个连续物理量的近似值。
\end{itemize}

这些任务的共同点是：目标 $y$ 不是离散标签，而是在一个连续区间内取值。因此，我们不再问“分类对了多少个”，而要问“预测值与真实值偏差多大”。

\section{回归的最小数学形式}

最简单的监督回归问题可以写成：

\[
\mathcal{D} = \{(x_i, y_i)\}_{i=1}^{N},
\]

其中 $x_i$ 是输入特征，$y_i$ 是连续目标。若使用参数化模型 $f_\theta(x)$，则回归的目标可以理解为：寻找参数 $\theta$，使得预测值 $\hat{y}_i = f_\theta(x_i)$ 与真实值 $y_i$ 尽可能接近。

在线性回归最简单的一维情形里，我们常写成：

\[
y = \beta_0 + \beta_1 x + \epsilon,
\]

其中：

\begin{itemize}
    \item $\beta_0$ 是截距；
    \item $\beta_1$ 是斜率；
    \item $\epsilon$ 表示噪声或未建模部分。
\end{itemize}

\begin{defbox}[回归任务的核心]
回归并不是“把一个数拟合出来”这么简单，而是在数据、误差和模型假设之间建立一个可解释的连续映射。
\end{defbox}

\section{一个教学版光度红移问题}

本章配套 notebook 使用一个小型 SDSS 风格教学数据文件 \nolinkurl{photometric_redshift_demo.csv}。每一行记录若干颜色指数和一个教学标签 \texttt{specz}。在真实科学中，光度红移是一个经典而重要的回归任务：你希望根据多波段测光，去逼近原本需要光谱才能更可靠获得的红移信息。

这个教学数据并不试图复现真实巡天中的全部复杂性，例如：

\begin{itemize}
    \item 训练集偏差；
    \item 模板依赖；
    \item 尘埃与颜色退化；
    \item 高红移样本稀缺；
    \item 仪器间系统差异。
\end{itemize}

它的目标更克制：帮助读者在一个可控规模上先看清回归流程的骨架。

\section{为什么先从一元线性回归开始}

在 notebook 中，我们先只用一个颜色特征 \texttt{g-r} 预测红移。这当然不是现实中最强的做法，但它在教学上有三个重要优势：

\begin{itemize}
    \item 输入和输出关系最容易可视化；
    \item 读者可以直接看懂斜率和截距在做什么；
    \item 后续任何更复杂的方法，都有了一个清楚的 baseline 参照。
\end{itemize}

\subsection{最小二乘思想}

若模型是

\[
\hat{y}_i = \beta_0 + \beta_1 x_i,
\]

则一种最自然的拟合目标，是让所有残差平方和尽可能小：

\[
S(\beta_0, \beta_1) = \sum_{i=1}^{N}
\left(y_i - \beta_0 - \beta_1 x_i\right)^2.
\]

这就是最小二乘思想。它的意义在于：大误差会被平方放大，因此模型会更努力地避免出现特别离谱的偏差。

\begin{examplebox}[最小线性回归]
\begin{lstlisting}[style=pythonstyle]
def fit_line(xs, ys):
    x_mean = sum(xs) / len(xs)
    y_mean = sum(ys) / len(ys)

    numerator = sum((x - x_mean) * (y - y_mean)
                    for x, y in zip(xs, ys))
    denominator = sum((x - x_mean) ** 2 for x in xs)

    slope = numerator / denominator
    intercept = y_mean - slope * x_mean
    return slope, intercept
\end{lstlisting}
\end{examplebox}

这段代码虽然很短，却把回归中的核心对象都展示出来了：输入特征、目标值、均值中心化、斜率和截距。

\section{训练误差和测试误差为什么都要看}

一个非常常见的误区是：只要训练误差低，就认为模型已经学得很好。其实训练误差只能说明模型在已见过的数据上拟合得怎样，并不能保证它在新样本上仍然有效。

因此，本章 notebook 会保留固定的教学版训练/测试切分。即便样本很小，这也能帮助你建立一个重要习惯：

\begin{itemize}
    \item 训练集用于拟合参数；
    \item 测试集用于近似评估泛化；
    \item 两者的差异会直接暴露过拟合风险。
\end{itemize}

\section{怎样评估回归结果}

对回归问题来说，单独说“预测对了多少个”没有意义，因为连续值几乎不会刚好完全相等。更自然的问题是：预测值平均差了多远。

\subsection{MAE：平均绝对误差}

\[
\mathrm{MAE} =
\frac{1}{N}\sum_{i=1}^{N}
\left|y_i - \hat{y}_i\right|.
\]

MAE 的优点是直观。它告诉你：平均而言，预测值和真实值差了多少。

\subsection{RMSE：均方根误差}

\[
\mathrm{RMSE} =
\sqrt{
\frac{1}{N}\sum_{i=1}^{N}
\left(y_i - \hat{y}_i\right)^2
}.
\]

RMSE 对大误差更敏感。因此，如果少数样本偏得特别远，RMSE 往往会明显高于 MAE。

\begin{examplebox}[一个最小 MAE 函数]
\begin{lstlisting}[style=pythonstyle]
def mean_absolute_error(y_true, y_pred):
    return sum(abs(a - b) for a, b in zip(y_true, y_pred)) / len(y_true)
\end{lstlisting}
\end{examplebox}

\section{残差：为什么不能只报指标}

设残差为

\[
r_i = y_i - \hat{y}_i.
\]

残差分析的价值在于，它把“一个整体误差数字”重新拆回到每个样本上。通过残差，你可以继续追问：

\begin{itemize}
    \item 模型是否总在低红移端高估、在高红移端低估；
    \item 某几个离群样本是否主导了 RMSE；
    \item 误差是否随着特征值增大而系统性放大；
    \item 某类样本是否超出训练集支持范围。
\end{itemize}

在科学任务里，这些问题通常比单一指标更接近真正的建模判断。

\section{为什么要引入多项式特征}

真实科学关系往往不是完美直线。颜色与红移之间、某些实验量与参数之间，都可能存在弯曲趋势。这时，我们会考虑把特征扩展成：

\[
1,\quad x,\quad x^2,\quad \dots
\]

对应的模型可以写成：

\[
\hat{y} = a_0 + a_1 x + a_2 x^2.
\]

多项式特征的价值在于：它让一个原本只能画直线的模型，开始有能力表达曲率。

\section{表达能力提高，为什么风险也会增加}

更多特征并不自动等于更好的科学模型。随着表达能力上升，模型可能开始：

\begin{itemize}
    \item 把训练集中的噪声也一并拟合进去；
    \item 在样本边界附近给出看似贴合、实际不稳的曲线；
    \item 在训练范围之外快速失真。
\end{itemize}

这就是过拟合最基础的直觉。教学上最重要的，不是记住术语，而是学会比较：

\begin{itemize}
    \item 训练误差是否继续下降；
    \item 测试误差是否真的同步改善；
    \item 模型外推是否开始变得不可信。
\end{itemize}

\section{为什么还要看更成熟的标准库实现}

配套 notebook 的可选部分会在环境可用时调用 \texttt{scikit-learn} 里的若干回归模型。这一步的目的，不是把手写实现抛弃掉，而是帮助读者理解：

\begin{itemize}
    \item 手写的一维线性回归 baseline 和标准工具链在思想上是连续的；
    \item 正则化与更复杂模型，是在 baseline 之上逐步增加控制力；
    \item 成熟生态让你更方便比较不同模型，但并不会替你自动完成科学判断。
\end{itemize}

\section{回归结果为什么不能脱离物理背景}

即使一个模型在测试集上 MAE 很小，也不能说明它已经“理解了物理”。特别要警惕以下情况：

\begin{itemize}
    \item 模型只在低红移样本上训练，却被拿去预测更高红移天体；
    \item 特征来自某台仪器，却被直接迁移到另一台仪器的观测上；
    \item 模型拟合成功的是经验关系，而不是机制方程；
    \item 数据覆盖范围太窄，导致所谓“规律”只是局部近似。
\end{itemize}

因此，本章回归流程最后仍然必须回到两个问题：

\begin{itemize}
    \item 这个误差大小对当前科学目标是否足够好；
    \item 这个模型是否被使用在它没有见过的区域。
\end{itemize}

\begin{warnbox}[回归中的常见误区]
\begin{itemize}
    \item 只看训练误差，不看测试误差；
    \item 把测试集反复拿来调参；
    \item 只报 MAE 或 RMSE，不检查残差结构；
    \item 超出训练样本覆盖范围仍然过度相信预测结果；
    \item 把经验回归关系直接当成物理定律。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在回归任务里，AI 助手适合帮助你：

\begin{itemize}
    \item 检查线性回归或多项式回归代码是否与定义一致；
    \item 帮你把 MAE、RMSE 和残差分析整理成更规范的报告；
    \item 对比 baseline 与更复杂模型的改进是否真实稳定；
    \item 提醒你哪些预测已经接近外推，应该谨慎解释。
\end{itemize}

但“这个模型是否满足科学精度要求”“这个误差是否可接受”“这个关系是否还能外推”，仍然需要你自己结合数据来源和科学目标判断。

\section{本章小结}

回归是天文和物理中最常见的机器学习任务之一，但它绝不是“把一串数拟合成一条线”那么简单。本章通过一个教学版光度红移问题，把连续目标、最小二乘、误差指标、残差分析、多项式特征和外推风险连成了一条完整工作流。只要这条链真正建立起来，你后面面对更高维、更非线性或更高精度要求的回归任务时，就不会只停留在“把模型跑出来”的层面。
