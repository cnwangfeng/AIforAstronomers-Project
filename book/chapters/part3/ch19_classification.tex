\chapter{分类}

\section{学习目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 理解分类任务与回归任务的根本差别，并知道科学中的分类标签往往只是物理结构的代理表达；
    \item 把一个最小分类问题写成“特征 $\rightarrow$ 决策边界 $\rightarrow$ 标签预测”的形式；
    \item 用一个教学数据集走通“规则 baseline $\rightarrow$ 数据驱动分类器 $\rightarrow$ 错误诊断”的完整链条；
    \item 理解 kNN、逻辑回归和决策树这几类经典分类器各自如何组织决策边界；
    \item 说明为什么混淆矩阵和按类别指标，往往比单一准确率更接近科学问题本身。
\end{itemize}

\section{为什么科学问题里经常需要分类}

很多科学任务不是预测一个连续量，而是判断一个对象更可能属于哪一类。例如：

\begin{itemize}
    \item 星、星系和类星体的粗分类；
    \item 光谱类型或目标类型的自动初筛；
    \item 信号事件与背景事件的区分；
    \item 暂现源候选体的优先级判别；
    \item 仪器伪迹与真实目标的快速区分。
\end{itemize}

这些任务的共同点是：输出不是一个连续实数，而是离散标签集合中的一个元素。因此，它们自然落在分类问题的框架里。

\section{分类任务的最小数学形式}

若样本特征记为 $x$，标签集合记为

\[
\mathcal{Y} = \{1,2,\dots,K\},
\]

则一个分类器的目标可以理解为：给定输入 $x$，输出一个预测标签

\[
\hat{y} = f(x) \in \mathcal{Y}.
\]

如果模型还能给出每一类的分数或概率估计，则常写成

\[
\hat{y} = \arg\max_{k \in \mathcal{Y}} \, s_k(x),
\]

其中 $s_k(x)$ 可以是某种打分函数、后验概率近似，或其他可比较的置信分值。

\begin{defbox}[分类的核心]
分类任务的关键，不是给每个对象硬塞一个标签，而是在特征空间里建立一条或一组边界，说明为什么这些对象应当被分开。
\end{defbox}

\section{标签并不总是“真相本身”}

进入科学分类时，必须保持一个清醒认识：标签往往是研究者为了组织问题而定义的代理结构，而不一定是自然界本身的绝对分箱。

例如：

\begin{itemize}
    \item “star / galaxy / quasar” 是观测层面非常有用的分类，但并不意味着每个样本都毫无模糊性；
    \item 某些目标可能处于边界区域，受信噪比、红移或仪器效应影响而表现混合特征；
    \item 教学数据里的标签通常比真实任务干净得多。
\end{itemize}

因此，分类模型学到的，从来都是“在当前标签体系下如何划分”，而不是宇宙自动告诉我们的唯一真理。

\section{一个教学版天体分类问题}

本章配套 notebook 使用 \nolinkurl{object_classification_demo.csv}。它包含若干对象的颜色指数，以及教学标签：

\begin{itemize}
    \item \texttt{star}；
    \item \texttt{galaxy}；
    \item \texttt{quasar}。
\end{itemize}

这个数据集不试图复现真实巡天中的全部复杂边界，而是为了把分类问题拆成最基本的几步：

\begin{itemize}
    \item 选择特征；
    \item 划分训练集与测试集；
    \item 建立规则 baseline；
    \item 训练一个最小数据驱动分类器；
    \item 用混淆矩阵和按类别指标看清错误分布。
\end{itemize}

\section{为什么先从规则 baseline 开始}

在正式引入数据驱动分类器前，我们先写一个简单规则分类器：

\begin{examplebox}[规则 baseline]
\begin{lstlisting}[style=pythonstyle]
def classify_rule(row):
    if row["u_g"] < 0.5 and row["g_r"] < 0.1:
        return "quasar"
    if row["g_r"] > 0.7 and row["r_i"] > 0.3:
        return "galaxy"
    return "star"
\end{lstlisting}
\end{examplebox}

这一步的意义，不是为了追求最高分数，而是为了建立一把清楚的尺子。它迫使我们先问：

\begin{itemize}
    \item 哪些颜色特征在物理上有区分作用；
    \item 阈值规则到底在表达什么观测直觉；
    \item 一个更复杂模型是否真的超越了这组直觉。
\end{itemize}

如果复杂模型只比这个 baseline 稍微好一点，那么问题也许根本还不值得进入更高复杂度。

\section{kNN 的直觉：边界由样本组织出来}

\textbf{k 最近邻（kNN）} 是本科教学中特别适合入门的一类分类器。它的思想非常直接：

\begin{enumerate}
    \item 把每个训练样本看成特征空间中的一个点；
    \item 对新样本，找出离它最近的 $k$ 个训练邻居；
    \item 用这些邻居的多数标签作为预测结果。
\end{enumerate}

这说明分类边界并不是凭空写出来的，而是由训练样本在空间中的相对位置自然组织出来的。

若训练集为 $\{(x_i, y_i)\}$，距离函数记为 $d(x, x_i)$，则 kNN 的直觉可以理解为：先找出最小化距离的邻居集合，再对邻居标签做投票。

\section{距离函数为什么也带着假设}

在 kNN 中，我们常用欧氏距离：

\[
d(x, x') =
\sqrt{
\sum_{j=1}^{p}
(x_j - x'_j)^2
}.
\]

但这个选择并不“中性”。它隐含地假设：

\begin{itemize}
    \item 各个特征维度可以直接比较；
    \item 特征尺度大致相容；
    \item 欧氏空间中的几何邻近，对标签也有意义。
\end{itemize}

这就是为什么特征工程和标准化在真实分类项目里如此重要。一个分类器不是只由“模型名字”决定的，距离定义本身就已经包含建模判断。

\section{逻辑回归：线性边界与概率分数}

\textbf{逻辑回归} 虽然名字里有“回归”，但在很多场合其实是经典分类器。对二分类问题，它通过线性组合

\[
z = \beta_0 + \beta^\top x
\]

构造一个分数，再通过 sigmoid 函数把它压到 $(0,1)$ 区间：

\[
P(y=1 \mid x) \approx \sigma(z) = \frac{1}{1 + e^{-z}}.
\]

它的优点是：

\begin{itemize}
    \item 决策边界容易解释；
    \item 参数和特征方向关系清楚；
    \item 对中小型结构化数据常常已经足够有力。
\end{itemize}

它的局限也很明确：如果真实边界高度非线性，逻辑回归就可能显得过于刚性。

\section{决策树：用规则切空间}

\textbf{决策树} 的直觉很像不断问一串问题：

\begin{itemize}
    \item \texttt{u-g} 是否小于某阈值；
    \item \texttt{g-r} 是否大于某阈值；
    \item 若满足某组合条件，进入哪个叶节点。
\end{itemize}

与逻辑回归相比，树模型更容易表达非线性边界与分段规则；但它也更容易在小样本上过度拟合。因此，分类器的选择往往是偏差、方差和可解释性之间的折中，而不是“谁更高级”。

\section{为什么准确率不够}

分类任务里最常见的总体指标之一是准确率：

\[
\mathrm{Accuracy} =
\frac{1}{N}
\sum_{i=1}^{N}
\mathbb{1}(\hat{y}_i = y_i).
\]

它确实直观，但也极其容易掩盖问题。例如，一个模型整体准确率不低，却可能总是在某一类上表现很差。对于科学任务来说，这种“局部失明”往往比总体分数更重要。

\section{混淆矩阵：看清错误结构}

混淆矩阵之所以关键，是因为它把“错了多少”重新拆成“错成了什么样”。例如：

\begin{examplebox}[一个最小混淆矩阵结构]
\begin{lstlisting}[style=pythonstyle]
{
    "star":   {"star": 2, "galaxy": 0, "quasar": 0},
    "galaxy": {"star": 1, "galaxy": 1, "quasar": 0},
    "quasar": {"star": 0, "galaxy": 0, "quasar": 2},
}
\end{lstlisting}
\end{examplebox}

这类结构能帮助我们回答：

\begin{itemize}
    \item 模型是不是系统性把某两类混淆；
    \item 哪一类最难识别；
    \item 某类错误在科学上是否特别昂贵。
\end{itemize}

因此，混淆矩阵不是附属图表，而是分类结果解释的核心对象。

\section{分类结果为什么必须回到样本分布解释}

即使一个分类器在测试集上表现不错，也不代表它可以直接用于真实巡天。需要警惕的情形包括：

\begin{itemize}
    \item 训练集没有覆盖真实观测中的全部对象类型；
    \item 训练集太小，边界高度依赖几个样本；
    \item 某些颜色特征受到观测误差、消光或仪器系统差异影响；
    \item 边界样本本身就难以用单一标签描述。
\end{itemize}

分类器并不是在真空中工作的。它对谁有效、在哪些样本上不稳、在哪些区域会失效，都必须放回数据来源和观测条件中解释。

\section{一个最小分类结果报告模板}

完成分类实验后，报告不应只写“准确率是多少”。一个更像科研记录的写法可以是：

\begin{quote}
本实验使用教学版颜色特征区分 \texttt{star}、\texttt{galaxy} 和 \texttt{quasar}，先以物理直觉规则作为 baseline，再比较 kNN、逻辑回归或决策树分类器。最终报告总体准确率、混淆矩阵和按类别 precision/recall，并逐一查看被错分样本是否集中在颜色边界、低信噪比或标签模糊区域。当前结果只说明模型在这个教学标签体系和当前切分下有效；真实巡天应用还需要重新检查训练集覆盖、观测误差、消光和类别定义。
\end{quote}

这段报告的核心是把“模型给了标签”改写成“模型在某个标签体系和样本覆盖范围内给出了可诊断的预测”。分类结果越要进入科学筛选，越不能省略错分类型和错误代价。

进入 capstone 时，这类分类记录会分别支撑 Baseline card、Evaluation card 和 Interpretation card：baseline 说明模型是否真的改进，混淆矩阵说明错误结构，错分样本复查则说明哪些标签边界仍需要人工判断。

\begin{warnbox}[分类中的常见误区]
\begin{itemize}
    \item 只看准确率，不检查哪一类最容易被错分；
    \item 训练集中根本没有覆盖某类对象，却仍然把预测结果当真；
    \item 在测试集上反复调规则，直到分数满意为止；
    \item 把教学样本上的高分直接误认为真实巡天可用的分类器；
    \item 看到“模型做对了”就误以为“模型理解了物理类型”。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在分类任务里，AI 助手适合帮助你：

\begin{itemize}
    \item 检查一个 baseline 规则是否自洽、是否与物理直觉一致；
    \item 帮你把 kNN、逻辑回归或混淆矩阵代码整理成更清楚的结构；
    \item 对比不同分类器在决策边界与可解释性上的取舍；
    \item 协助你把“哪些样本被错分了”写成更像科学分析而不是成绩单的说明。
\end{itemize}

但分类边界是否合理、哪些错误最不可接受、模型是否还能迁移到新观测条件，这些判断仍然必须由你自己结合科学目标完成。

\section{练习}

\begin{exercisebox}[基础练习]
解释为什么一个三分类模型即使准确率相同，也可能因为“错分模式不同”而在科学上价值完全不同。
\end{exercisebox}

\begin{exercisebox}[进阶练习]
从欧氏距离公式出发，说明为什么特征尺度不一致时，kNN 的结果可能会被某一维特征主导。
\end{exercisebox}

\begin{exercisebox}[开放练习]
为一个你感兴趣的科学分类任务写一页说明：标签是什么、特征是什么、baseline 是什么、最担心哪种错分，以及为什么不能只报准确率。
\end{exercisebox}

\section{本章小结}

分类问题的核心，不是挑一个模型名字，而是在特征空间中建立能被解释、能被诊断的决策边界。本章通过一个教学版天体分类例子，把规则 baseline、kNN、逻辑回归、决策树和混淆矩阵串成了一条完整链条。只要这条链真正建立起来，后面你在面对更高维特征、更复杂模型或更稀有目标检索任务时，才不会把“会输出标签”误当成“会做科学分类”。
