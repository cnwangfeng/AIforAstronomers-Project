\chapter{模型评估与诊断}

\section{学习目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 解释为什么模型评估不能停留在单一分数，尤其不能只盯着准确率；
    \item 读懂混淆矩阵、precision、recall、F1 在科学任务中的具体含义；
    \item 理解阈值选择、ROC 曲线和 PR 曲线为什么本质上是在表达不同错误代价之间的取舍；
    \item 知道交叉验证是在估计稳定性，而不是替代最终测试集；
    \item 理解校准、类别不平衡和科学代价函数为什么会决定一个模型是否真正可用。
\end{itemize}

\section{为什么“分数不错”远远不够}

初学机器学习时，最常见的误区之一，就是把一个看起来不错的准确率或单个分数，当成模型已经可靠的证据。但在真实科学问题里，我们往往还会关心：

\begin{itemize}
    \item 模型最容易错分的是哪一类；
    \item 漏掉一个重要目标和误报一个普通目标，代价是否相同；
    \item 这个结果是不是只在某个幸运切分上好看；
    \item 模型输出的“0.9”到底能不能当成九成概率来解释。
\end{itemize}

因此，模型评估不是训练完成后附带填上的一栏，而是整个工作流中最决定可信度的一环。

\section{一个教学版“类星体检索”任务}

为了把评估概念讲得更清楚，本章把教学数据
\nolinkurl{object_classification_demo.csv}
暂时改写成一个一对多问题：

\begin{itemize}
    \item 正类：\texttt{quasar}；
    \item 负类：\texttt{star} 与 \texttt{galaxy}。
\end{itemize}

这样做不是因为真实科学问题只会做二分类，而是因为 precision、recall、阈值和 ROC/PR 的核心直觉，在二分类设置里最容易建立。

\section{混淆矩阵：先把错误拆开}

评估二分类器时，我们首先关心四种结果：

\begin{itemize}
    \item \textbf{TP}（true positive）：真正的类星体被正确找出；
    \item \textbf{FN}（false negative）：真正的类星体被漏掉；
    \item \textbf{FP}（false positive）：普通目标被误报成类星体；
    \item \textbf{TN}（true negative）：普通目标被正确排除。
\end{itemize}

混淆矩阵正是把这四种结果结构化记录下来：

\begin{examplebox}[二分类混淆矩阵]
\begin{lstlisting}[style=pythonstyle]
{
    "positive": {"positive": TP, "negative": FN},
    "negative": {"positive": FP, "negative": TN},
}
\end{lstlisting}
\end{examplebox}

这一结构的价值在于：它不只是告诉我们“错了多少”，而是告诉我们“错成了什么样”。

\section{准确率为什么会掩盖问题}

准确率的定义是

\[
\mathrm{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}.
\]

它的优点是直观，但也很容易掩盖问题。举例来说，若类星体本来就是稀有目标，那么一个总是预测“非类星体”的模型，也可能得到不低的准确率。可从科学角度看，这样的模型几乎没有价值，因为它把所有真正需要发现的对象都漏掉了。

因此，在不平衡任务里，准确率往往只是一个粗略总体指标，远不够支撑科学决策。

\section{Precision 与 Recall：两种完全不同的关切}

\subsection{Precision}

\[
\mathrm{Precision} = \frac{TP}{TP + FP}.
\]

precision 回答的问题是：\textbf{你报出来的候选里，有多少是真的。}

如果后续要做昂贵的光谱跟踪或望远镜申请，那么 precision 往往非常重要，因为每个误报都会消耗真实资源。

\subsection{Recall}

\[
\mathrm{Recall} = \frac{TP}{TP + FN}.
\]

recall 回答的问题是：\textbf{真正存在的目标里，你找回了多少。}

如果你的科学目标是尽可能不漏掉稀有目标，那么 recall 往往比 precision 更重要。

\subsection{F1}

\[
F_1 =
\frac{2 \cdot \mathrm{Precision} \cdot \mathrm{Recall}}
{\mathrm{Precision} + \mathrm{Recall}}.
\]

F1 尝试在 precision 和 recall 之间做一个平衡，但它仍然只是压缩后的单个数字。真正重要的，仍然是你理解这两者之间的科学张力。

\section{阈值不是技术细节，而是科学决策}

很多分类模型输出的不是直接标签，而是一个分数或概率近似。只要改变阈值，precision 和 recall 往往就会一起变化：

\begin{itemize}
    \item 阈值更高：模型更保守，误报往往减少，但漏报增加；
    \item 阈值更低：模型更激进，召回往往提高，但误报也会上升。
\end{itemize}

这说明阈值不是小参数，而是把模型输出转化为科学行动时必须做出的选择。

\begin{protip}[阈值本质上编码了错误代价]
如果你最怕误报，阈值通常应该更高；如果你最怕漏掉稀有目标，阈值通常应该更低。阈值设置反映的是科学代价函数，而不是代码习惯。
\end{protip}

\section{ROC 与 PR 曲线：两种不同的观察角度}

\subsection{ROC 曲线}

ROC 曲线关注真阳性率（TPR）和假阳性率（FPR）之间的关系，其中

\[
\mathrm{TPR} = \frac{TP}{TP + FN},
\qquad
\mathrm{FPR} = \frac{FP}{FP + TN}.
\]

随着阈值变化，一个分类器会在 ROC 平面上扫出一条曲线。它帮助我们观察：为了得到更高召回，需要付出多大的假阳性代价。

\subsection{PR 曲线}

PR 曲线则直接把 precision 和 recall 放到同一平面上。对于稀有目标检索、暂现源筛选等不平衡任务，PR 曲线往往更贴近科学直觉，因为它更直接反映“找回多少”和“报出来的有多纯”之间的交换。

\section{为什么交叉验证是在评估稳定性}

一次训练/测试切分，只能告诉我们“在这一次切分上模型表现如何”。若数据量不大，这个结果往往对样本划分非常敏感。

交叉验证的作用是：

\begin{itemize}
    \item 把数据拆成多个折；
    \item 在不同切分上重复训练与评估；
    \item 观察分数的均值和波动。
\end{itemize}

这实际上是在回答一个更深的问题：\textbf{这个模型的好坏，是不是稳定的，还是只依赖某一次幸运切分。}

但要特别强调：交叉验证不是最终测试集的替代品。最终测试集仍应保留到最后，用于给出独立报告结果。

\section{校准：会排序，不等于会报概率}

一个模型给某个样本打了 $0.9$ 分，并不自动意味着“它有九成概率是真的”。这正是校准要回答的问题：模型分数是否真的对应了现实中的频率。

一个模型可能：

\begin{itemize}
    \item 排序能力很好，能把真正的目标排到前面；
    \item 但分数尺度并不可靠，0.9 不一定真的是九成概率。
\end{itemize}

在许多科学任务里，这一点非常关键。因为你后续是否做跟踪观测、是否提高优先级、是否作为高置信候选发布，往往都会依赖这个分数解释。

\section{Brier score：一个最小校准视角}

若模型输出概率近似 $p_i$，真实标签为 $y_i \in \{0,1\}$，则一个简单的概率质量指标是 Brier score：

\[
\mathrm{Brier} =
\frac{1}{N}\sum_{i=1}^{N}(y_i - p_i)^2.
\]

它同时对排序与概率偏差敏感。教学上最重要的，不是立刻把它神化，而是通过它建立一种意识：\textbf{概率输出本身也需要被检验。}

\section{类别不平衡与科学代价函数}

很多科学目标天生就是稀有的，例如暂现源、异常目标、强透镜候选、某些特殊光谱对象。此时，平均分数很容易掩盖真正重要的问题。

你必须明确：

\begin{itemize}
    \item 漏掉一个罕见目标和误报一个普通目标，哪个代价更高；
    \item 后续观测资源是否昂贵；
    \item 一个错误分类会不会在样本统计上引入系统偏差；
    \item 你的模型到底是为“高纯度候选列表”服务，还是为“高召回预筛选”服务。
\end{itemize}

这说明评估指标不是独立于科学目标存在的。指标的解释，必须服从你的任务目的。

\section{诊断不只是数值，还要回到样本}

一个成熟的评估流程，不会停留在表格数字上，而会进一步问：

\begin{itemize}
    \item 哪些样本总是在阈值附近摇摆；
    \item 哪些误报样本看起来其实像边界对象；
    \item 哪些漏报样本可能提示特征设计不足；
    \item 高分样本是否真的对应高质量候选。
\end{itemize}

这一步就是“诊断”的含义。它要求我们从总体指标重新回到具体样本和具体错误模式。

\section{一个最小评估报告模板}

当你把模型评估写进 notebook 或项目报告时，不要只写“accuracy 很高”或“F1 不错”。这样的句子既不能复查，也不能告诉读者这个模型在科学任务中是否真的可用。一个最小评估报告至少应交代五件事：

\begin{enumerate}
    \item \textbf{任务定义}：正类和负类是什么，样本是否不平衡；
    \item \textbf{数据切分}：训练、验证、测试如何划分，测试集是否最后才使用；
    \item \textbf{核心指标}：报告 accuracy 之外，还要报告 precision、recall、F1 或任务相关指标；
    \item \textbf{阈值理由}：说明阈值对应的是高纯度候选、还是高召回预筛选；
    \item \textbf{错误诊断}：列出主要误报、漏报类型，并说明它们对后续科学行动的影响。
\end{enumerate}

\begin{examplebox}[评估报告的最小骨架]
\begin{lstlisting}[style=pythonstyle]
task = "quasar retrieval"
positive_class = "quasar"
split_note = "train/validation/test fixed before tuning"
threshold_policy = "prefer high recall for candidate screening"
report = {
    "accuracy": accuracy,
    "precision": precision,
    "recall": recall,
    "f1": f1,
    "main_false_positives": fp_examples,
    "main_false_negatives": fn_examples,
}
\end{lstlisting}
\end{examplebox}

真正好的评估报告不是指标越多越好，而是每个指标都能回答一个清楚的科学问题。若指标、阈值和错误代价没有连起来，模型分数就仍然停留在竞赛式描述，而没有进入科研证据链。

\begin{warnbox}[评估中的典型误区]
\begin{itemize}
    \item 只报告准确率，不报告按类别指标；
    \item 在测试集上反复调阈值直到分数好看；
    \item 把交叉验证均值直接当成最终测试结果；
    \item 把未校准的分数直接当作概率解释；
    \item 忽视类别不平衡，导致真正的科学目标被“平均分数”掩盖。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这一章，AI 助手适合帮助你：

\begin{itemize}
    \item 检查 precision、recall、F1、Brier score 等指标代码是否与定义一致；
    \item 帮你解释某个混淆矩阵或阈值扫描表暗示了什么错误模式；
    \item 对比 ROC 和 PR 在不平衡任务中的适用场景；
    \item 协助你把“评估结果”整理成更像科学判断而不是单纯竞赛分数的说明。
\end{itemize}

但阈值到底该如何设置、哪类错误代价更高、最终该报告哪些指标，这些判断仍然必须服从你的科学目标，而不是由模型或 AI 助手替你决定。

\end{exercisebox}

\begin{exercisebox}[开放练习]
为一个你感兴趣的科学检索任务写一页评估方案，至少包含：正负类定义、最关心的错误类型、阈值策略、主要指标、是否需要校准和交叉验证。
\end{exercisebox}

\section{本章小结}

模型评估的真正目标，不是给结果配一个好看的分数，而是判断它在你的科学任务里是否可信、稳定、可解释、可行动。本章通过一个教学版类星体检索问题，把混淆矩阵、precision/recall、阈值曲线、交叉验证、校准和类别不平衡放进同一套判断框架里。只要这套框架真正建立起来，后面你面对更复杂模型时，就不会把“模型得分高”误当成“模型已经可以被科学信任”。
