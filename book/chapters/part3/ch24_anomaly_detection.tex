\chapter{异常检测与新奇发现}

\section{本章导读：异常分数只是候选排序，不是发现宣言}

异常检测在天文学里格外吸引人，因为“异常”很容易让人联想到新天体、新现象或新物理。但真实项目中，高异常分数更常见的来源可能是坏点、缺失值、边界样本、预处理失败、仪器差异或训练集覆盖不足。把异常分数直接当作发现，是这一类任务最大的陷阱。

本章把异常检测讲成候选排序和复核流程，而不是最终判决。距离方法、LOF、Isolation Forest 和 autoencoder 各自有不同假设，因此它们排序不同很正常。真正重要的是：候选为什么被排到前面，它是否在数据质量上可疑，是否需要人工复核，是否能被独立证据支持。

读这一章时，请把每个异常候选放进三层结论里：坏数据、边界样本、值得后续检查的科学候选。只有经过这种分层，异常检测才不会变成“模型说它怪，所以它重要”。

\section{学习目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 解释异常检测与普通分类任务在目标上的根本差别；
    \item 理解基于距离、LOF 和 Isolation Forest 各自抓住了哪类“异常”；
    \item 明白 autoencoder 更关注重构误差，而不是直接给出物理解释；
    \item 认识到异常分数更适合做候选排序，而不是自动宣布“新发现”；
    \item 建立人工复核、数据质量排查和后续观测验证的完整意识。
\end{itemize}

\section{异常点不一定是新物理，也可能是坏数据}

在科学数据里，“异常”这个词很迷人，因为它总让人想到新现象、新天体或新机制。但在真实工作中，异常点通常至少可能来自三种来源：

\begin{itemize}
    \item \textbf{真正稀有的物理对象}，例如特殊光谱源、极端变量源；
    \item \textbf{观测或处理伪影}，例如坏像元、减天光失败、宇宙线污染；
    \item \textbf{样本边缘但并不错误的普通对象}，只是它们在当前数据集中比较少见。
\end{itemize}

因此，异常检测真正擅长的，不是“宣布发现”，而是\textbf{把最值得你先看一眼的样本排到前面。}

\section{异常检测与监督分类的差别}

在监督分类里，我们通常已有明确标签，模型学习的是“如何把已知类别分开”。而在异常检测里，我们往往面对的是：

\begin{itemize}
    \item 没有完整标签；
    \item 稀有目标非常少；
    \item 甚至连“异常到底是什么”都没有统一定义。
\end{itemize}

因此，异常检测更像是在问：

\begin{itemize}
    \item 这个点离主体结构有多远；
    \item 它所在局部是否显得特别稀疏；
    \item 它是否很容易被从总体里单独隔离出来；
    \item 它是否难以被“正常样本模型”重构。
\end{itemize}

这说明异常检测通常输出的不是“最终标签”，而是一个排序分数、候选优先级或复核清单。

\section{一个教学版光谱特征例子}

本章配套 notebook 使用 \nolinkurl{spectral_anomaly_demo.csv}。它不是完整光谱，而是从光谱里抽取出的几个教学特征：

\begin{itemize}
    \item \texttt{halpha\_ew}：H$\alpha$ 等效宽度；
    \item \texttt{caii\_k\_ew}：Ca II K 吸收强度；
    \item \texttt{blue\_red\_ratio}：蓝端与红端连续谱比值；
    \item \texttt{snr}：信噪比。
\end{itemize}

在这组数据里，大多数样本形成两团正常恒星结构；另外有三类刻意设计的异常候选：

\begin{itemize}
    \item 发射线候选；
    \item 热致密天体候选；
    \item 一个光谱处理伪影。
\end{itemize}

关键在于：\textbf{算法并不知道哪一个是“真实异常”，它只知道某些点和主体分布不一样。}

\section{最朴素的起点：离邻居有多远}

最简单的异常分数往往来自一个非常直观的问题：一个样本和它最近的几个邻居相比，到底有多远？

\begin{examplebox}[距离型异常分数]
\begin{lstlisting}[style=pythonstyle]
anomaly_score = mean(distance to k nearest neighbors)
\end{lstlisting}
\end{examplebox}

这种方法的优点是：

\begin{itemize}
    \item 直觉强；
    \item 实现简单；
    \item 在主体结构比较清楚时常常很有效。
\end{itemize}

但它也有明显局限：如果数据本来就有多个正常群体，单纯看距离可能会把“小众但正常”的局部簇也误判成异常。

\section{本章算法卡片的最小问题}

异常检测算法的差异，首先来自它们对“异常”的定义不同。因此每个 Algorithm Card 至少要回答：它认为怎样的点异常？

\begin{itemize}
    \item \textbf{Distance-based anomaly}：认为离最近邻或局部邻居很远的点更异常；要检查标准化和多正常群体风险；
    \item \textbf{LOF}：认为局部密度显著低于邻居区域的点更异常；要检查邻居数和局部密度尺度；
    \item \textbf{Isolation Forest}：认为容易被随机切分单独隔离的点更异常；要检查随机性、污染率和高维特征含义；
    \item \textbf{Autoencoder reconstruction error}：认为难以被正常样本表示重构的点更异常；要检查模型容量、训练集污染和重构误差解释边界。
\end{itemize}

这些卡片不需要把算法推导成完整数学教材，但必须让学生知道：异常分数不是统一真理，而是某一种异常定义下的候选排序。

\begin{examplebox}[Algorithm Card: Distance-based Anomaly]
\begin{itemize}
    \item \textbf{任务类型}：异常候选排序；
    \item \textbf{决策规则}：离最近邻或局部邻居越远，异常分数越高；
    \item \textbf{用户固定}：距离度量、邻居数 \(k\)、标准化方式和参考样本；
    \item \textbf{诊断}：最近邻是谁、触发距离的主要特征、是否存在多个正常群体；
    \item \textbf{失败模式}：尺度主导距离，小众但正常的群体被误判为异常；
    \item \textbf{科学边界}：能排序“远离主体”的点，不能说明远离原因一定是新物理。
\end{itemize}
\end{examplebox}

\begin{examplebox}[Algorithm Card: Local Outlier Factor]
\begin{itemize}
    \item \textbf{任务类型}：局部异常检测；
    \item \textbf{决策规则}：若一个点的局部密度显著低于其邻居区域，异常分数升高；
    \item \textbf{用户固定}：邻居数、距离度量、标准化方式；
    \item \textbf{诊断}：局部邻居、局部密度对比、边界样本是否集中；
    \item \textbf{失败模式}：邻居数选择不当会把正常稀疏区或边界区误报；
    \item \textbf{科学边界}：能发现局部孤立样本，不能自动区分物理稀有与数据质量问题。
\end{itemize}
\end{examplebox}

\begin{examplebox}[Algorithm Card: Isolation Forest]
\begin{itemize}
    \item \textbf{任务类型}：异常候选排序；
    \item \textbf{决策规则}：越容易被随机特征切分单独隔离的样本，越像异常；
    \item \textbf{用户固定}：树数量、子样本大小、污染率或候选比例、随机种子；
    \item \textbf{诊断}：高分样本触发哪些特征、不同随机种子下排序是否稳定；
    \item \textbf{失败模式}：高维无意义特征会制造隔离假象，污染率设置会影响候选数量；
    \item \textbf{科学边界}：适合大样本快速排序，不提供异常原因的物理解释。
\end{itemize}
\end{examplebox}

\begin{examplebox}[Algorithm Card: Autoencoder Reconstruction Error]
\begin{itemize}
    \item \textbf{任务类型}：表示学习式异常检测；
    \item \textbf{模型形式}：编码器把输入压缩到低维表示，解码器再重构输入；
    \item \textbf{优化目标}：最小化训练样本的重构误差；
    \item \textbf{用户固定}：网络容量、瓶颈维度、训练集定义、重构误差阈值；
    \item \textbf{诊断}：误差集中在哪些波段或特征，训练集是否被异常污染；
    \item \textbf{失败模式}：模型太强会记住异常，模型太弱会把正常复杂结构也重构不好；
    \item \textbf{科学边界}：重构差只说明模型表示不了该样本，不等于已经发现新物理。
\end{itemize}
\end{examplebox}

\section{四种异常分数其实在问四个不同问题}

异常检测最容易被误解成“哪种算法更会发现异常”。更准确地说，不同算法在问不同问题：

\begin{itemize}
    \item \textbf{距离方法}：这个点离最近的正常邻居远不远？
    \item \textbf{LOF}：这个点所在位置是否比邻居所在区域更稀疏？
    \item \textbf{Isolation Forest}：随机切空间时，这个点是否很快被单独分出来？
    \item \textbf{Autoencoder}：用正常样本学到的低维表示，能否重构这个点？
\end{itemize}

因此，同一个样本在四种方法里排名不同，并不是坏事。它提醒你：这个样本到底是“远”、是“局部孤立”、是“容易被切出来”，还是“难以重构”。异常报告如果能写出触发原因，就比只给一个总分更接近科学复核。

\section{距离分数为什么必须先标准化}

本章的光谱特征量纲并不相同：等效宽度、连续谱比值和信噪比的数值范围完全不同。如果直接计算欧氏距离，距离最大的维度会主导异常分数。一个高信噪比差异很大的普通样本，可能会被排在真正光谱异常之前。

因此，最小距离型异常流程应写成：

\begin{enumerate}
    \item 先审查特征是否有物理意义；
    \item 用训练或参考主体样本估计标准化尺度；
    \item 在标准化空间中计算邻居距离；
    \item 把高分样本排序，而不是直接判定为新发现。
\end{enumerate}

\begin{examplebox}[标准化后的距离分数骨架]
\begin{lstlisting}[style=pythonstyle]
scaled_features = standardize(reference_features, all_features)
neighbors = find_k_nearest_neighbors(scaled_features, k=5)
score = mean_neighbor_distance(neighbors)
\end{lstlisting}
\end{examplebox}

这里的 \texttt{reference\_features} 很重要。若你把明显异常点也用于定义标准化尺度，异常本身就会反过来改变“正常”的坐标尺。

\section{LOF：判断你所在区域是不是太稀疏}

\textbf{Local Outlier Factor（LOF）} 的思路，是把一个点的局部密度和邻居的局部密度做比较。如果你周围明显比邻居周围更稀疏，那么你的 LOF 分数就会上升。

它特别适合处理这种情况：

\begin{itemize}
    \item 数据有多个正常群体；
    \item 不同区域的密度不完全一样；
    \item 你想找的是“局部上显得格格不入”的样本。
\end{itemize}

这类方法强调的不是“离整体中心有多远”，而是“和你最近的人相比，你是不是过于孤立”。

\paragraph{LOF 的最小直觉}

可以把 LOF 想成一个密度比值：

\[
\mathrm{LOF}(x)
\approx
\frac{\text{邻居附近的典型密度}}
{\text{\(x\) 附近的局部密度}}.
\]

如果这个比值接近 \(1\)，说明 \(x\) 和邻居处在差不多的密度环境；如果明显大于 \(1\)，说明 \(x\) 比周围邻居更孤立。这个直觉也解释了为什么邻居数很关键：邻居数太小，LOF 容易受单个点影响；邻居数太大，局部异常又会被全局结构平均掉。

\section{Isolation Forest：看一个点有多容易被单独切出来}

\textbf{Isolation Forest} 背后的直觉很漂亮：如果一个样本真的很孤立，那么用随机切分特征空间时，它往往会比较快被单独隔离出来。

这类方法的特点是：

\begin{itemize}
    \item 不必显式估计整体密度；
    \item 对高维数据通常更友好；
    \item 更像是在问“一个点被单独拎出来需要几刀”。
\end{itemize}

在教学 notebook 里，我们用一个简化版的随机切分深度来模拟这一思想。你会看到：处理伪影往往最容易被隔离，而两个物理异常候选也会得到较高分数，但三者的排序并不完全一样。

\paragraph{路径长度的直觉}

Isolation Forest 中，每棵树都会随机选择特征和切分值，把样本逐步分开。一个普通样本通常混在密集区域里，需要多次切分才会被单独隔离；一个孤立样本往往几次切分后就独自落到叶节点。平均路径长度越短，异常分数越高。这个机制让它适合大样本快速排序，但也意味着：如果某些无意义特征让样本显得孤立，它同样会被排到前面。

\section{不同异常方法为什么会给出不同排序}

同一批候选在不同算法下排序不同，并不一定说明算法“谁对谁错”，更可能说明它们定义的异常不同：

\begin{itemize}
    \item 距离方法强调离邻居远；
    \item LOF 强调局部密度比邻居稀疏；
    \item Isolation Forest 强调容易被随机切分隔离；
    \item autoencoder 强调难以被正常样本表示重构。
\end{itemize}

因此，异常检测报告应记录“触发异常的理由”。如果一个样本只在某一种方法里高分，而其他方法都不支持，它仍然值得看，但结论应更谨慎。如果多个定义都把它排在前列，它通常更值得优先人工复核。

\section{Autoencoder：重构不好的点值得多看一眼}

\textbf{Autoencoder} 常被用于异常检测，因为它试图先把数据压缩到一个低维瓶颈，再从瓶颈重构回原输入。如果训练主要见到的是普通样本，那么非常不寻常的对象往往会重构得比较差，于是重构误差升高。

但这里必须特别小心：

\begin{itemize}
    \item 高重构误差不自动等于新物理；
    \item 模型容量太大时，连异常也可能被“记住”；
    \item 训练集污染会直接改变你对“正常”的定义。
\end{itemize}

因此，在本科阶段更重要的是先理解 autoencoder 的\textbf{逻辑角色}，而不是把它当成神秘黑箱。

\section{异常检测更像候选排序，而不是最终判决}

在科研流程里，异常分数最常见的用途是：

\begin{itemize}
    \item 从几十万条记录里先筛出最值得人工看图的前几百个；
    \item 优先安排后续光谱检查、图像复核或交叉匹配；
    \item 在数据清洗阶段及早发现管线问题。
\end{itemize}

这意味着一个成熟的异常检测流程通常包含两步：

\begin{enumerate}
    \item \textbf{排序}：让算法给出候选优先级；
    \item \textbf{复核}：结合原始图像、原始光谱、观测条件和领域知识判断原因。
\end{enumerate}

\section{为什么人工复核不可省略}

如果你只看异常分数，很容易把“稀有而真实”与“糟糕但无意义”混在一起。人工复核至少要回答这些问题：

\begin{itemize}
    \item 原始数据质量是否可靠；
    \item 是否存在坏像元、饱和、背景扣除失败或管线拼接错误；
    \item 这个样本在其他波段、其他巡天或其他时刻是否一致；
    \item 它是新奇物理对象，还是只是训练集里不常见的已知类型。
\end{itemize}

真正的“新奇发现”往往诞生在这一步之后，而不是诞生在排行榜本身。

\section{一个最小异常候选复核表}

异常检测的交付物不应该只是一个按分数排序的表，而应该是一个可以进入人工复核的候选清单。每个高分候选至少需要记录：

\begin{enumerate}
    \item \textbf{候选身份}：样本编号、原始观测来源和基本质量标记；
    \item \textbf{异常证据}：异常分数、排名、触发该分数的主要特征或波段；
    \item \textbf{质量排查}：是否存在低信噪比、坏像元、饱和、拼接错误或背景扣除问题；
    \item \textbf{外部交叉检查}：其他波段、其他巡天或其他历元是否支持这个异常；
    \item \textbf{复核结论}：暂定为物理候选、数据伪影、边界普通样本，还是需要更多信息。
\end{enumerate}

\begin{examplebox}[异常候选复核记录]
\begin{lstlisting}[style=pythonstyle]
candidate_review = {
    "object_id": object_id,
    "rank": rank,
    "anomaly_score": score,
    "trigger_features": ["halpha_ew", "blue_red_ratio"],
    "quality_flags_checked": True,
    "cross_match_status": "needs_follow_up",
    "review_label": "candidate_not_confirmed",
}
\end{lstlisting}
\end{examplebox}

这样记录的好处是，它把“算法觉得奇怪”和“科学上值得相信”隔开了。前者可以自动化，后者必须经过证据、质量和后续验证共同支撑。

这份复核表是异常检测进入 Part III Model Experiment Packet 的关键接口：异常分数进入 evaluation artifact，质量排查和交叉检查进入 error analysis artifact，最后的复核结论进入 Trust Statement。没有这层记录，异常检测很容易把“算法觉得奇怪”误写成“科学上新奇”。

\section{Anomaly Review Triage：候选复核的分层结论}

人工复核后，最好不要只写“是异常”或“不是异常”。更有用的是把候选分成几层：

\begin{itemize}
    \item \textbf{数据伪影}：异常来自坏像元、低信噪比、背景扣除或拼接问题；
    \item \textbf{边界普通样本}：对象真实但只是当前样本中少见，不需要新物理解释；
    \item \textbf{物理候选}：有多个特征支持不寻常解释，但仍需外部验证；
    \item \textbf{待补信息}：当前证据不足，需要更多波段、历元或人工检查。
\end{itemize}

这种分层比二元标签更接近科研实际。异常检测的主要价值，是把注意力排序；确认和命名仍然依赖证据链。

本章的 Pass 要求是：不能只给 anomaly score；必须给人工复核字段和候选分层。最低限度也要回答三问：

\begin{enumerate}
    \item Bad data / artifact?
    \item Boundary but ordinary sample?
    \item Scientifically interesting candidate?
\end{enumerate}

\section{一个容易被忽略的问题：你到底在找什么异常}

异常检测常常被误解成一个统一问题，但实际上“异常”可能有很多含义：

\begin{itemize}
    \item 与整体分布很远的全局异常；
    \item 在局部邻域里不合群的局部异常；
    \item 在某个时间窗口里突然偏离的时序异常；
    \item 只在某些组合特征上显得奇怪的条件异常。
\end{itemize}

因此，方法选择必须回到你的科学目标，而不是看到哪个算法流行就直接套上去。

\section{阈值和污染率为什么也带着判断}

在很多异常检测流程中，最终还会遇到一个实际问题：到底选前多少个候选、或把多高的分数当作需要复核的阈值。这个决定往往对应于所谓的“污染率”假设，即你默认数据中大约有多少比例是不寻常样本。

如果阈值太低：

\begin{itemize}
    \item 你会得到很多误报；
    \item 人工复核成本会迅速上升。
\end{itemize}

如果阈值太高：

\begin{itemize}
    \item 你可能会漏掉真正稀有但不那么极端的目标；
    \item 算法排序的价值会被过度压缩。
\end{itemize}

因此，异常检测的阈值设置，本质上也是观测资源、人工成本和科学目标之间的平衡。

\begin{warnbox}[异常检测中的常见误区]
\begin{itemize}
    \item 把高异常分数直接当作“新发现”；
    \item 在没有标准化和特征审查的情况下直接跑算法；
    \item 忽视数据质量问题，把伪影当作天体物理异常；
    \item 用同一套阈值处理完全不同密度和噪声水平的数据；
    \item 只展示最戏剧化的异常图，而不报告人工复核后的淘汰率。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这一章，AI 助手适合帮助你：

\begin{itemize}
    \item 检查 LOF 或 Isolation Forest 的实现流程是否合理；
    \item 帮你整理异常候选表格与人工复核记录模板；
    \item 讨论某个高分样本更像物理异常还是数据质量问题；
    \item 协助你把“算法排序结果”和“人工复核结论”分成两层清楚记录。
\end{itemize}

但哪些特征用于排序、人工复核时先查什么、某个候选是否值得后续观测，这些决定仍然必须由你自己负责。

\section{练习}

\begin{exercisebox}[基础练习]
解释为什么一个在异常分数榜单上排名很高的样本，仍然完全可能只是数据质量问题。
\end{exercisebox}

\begin{exercisebox}[进阶练习]
比较“与整体分布很远”和“在局部邻域里很孤立”这两种异常定义，并说明为什么它们可能给出不同的候选排序。
\end{exercisebox}

\begin{exercisebox}[开放练习]
为一个你感兴趣的数据集写一页异常检测方案，至少包含：异常定义、候选特征、排序方法、人工复核步骤和后续验证方式。
\end{exercisebox}

\section{本章小结}

异常检测的价值，不在于替你自动宣布发现，而在于把有限注意力集中到最值得先看的对象上。距离分数、LOF、Isolation Forest 和 autoencoder 关注的是不同意义上的“异常”；而真正可靠的科研流程，一定还要把人工复核、数据质量排查和后续验证接回来。只有这样，异常排行榜才会变成科学候选清单，而不是误报制造机。
