\chapter{不确定度、解释性与科学可信度}

\section{本章导读：可信度不是模型最后贴上的标签}

到这一章时，我们已经能定义任务、训练模型、评估分数、调参、做预处理、探索结构和筛选异常。但这些仍然不足以让模型输出自动变成科学结论。真正的问题是：这个预测在多大范围内可信，它依赖了哪些信息，遇到训练覆盖范围之外的样本会怎样失败。

不确定度、解释性和可信度经常被当成“模型做完之后再补的解释”。本章要反过来讲：可信度应该从任务定义、数据切分、特征工程、评估设计和错误分析一路积累出来。bootstrap 区间、特征重要性、局部解释和覆盖范围检查，都只是这条链上的不同证据。

读这一章时，请格外警惕两种误读：把解释性当成因果证明，把不确定度数字当成完整风险。可信的机器学习结果，必须同时说明它为什么可能对、在哪里可能错、错了之后应该如何降级使用。

\section{学习目标}

学完本章后，你应该能够：

\begin{itemize}
    \item 理解为什么一个单独的预测值不足以支撑科学结论；
    \item 区分 bootstrap 区间、模型误差和数据分布外失效各自描述的是什么风险；
    \item 理解特征重要性和局部解释回答的是“模型如何用信息”，而不是“物理规律是什么”；
    \item 知道 SHAP 更像一种归因框架，而不是“证明某特征真实决定了结果”的证据；
    \item 建立一个基本判断：机器学习输出只有在误差、覆盖范围和选择效应一起被检查后，才谈得上可信。
\end{itemize}

\section{为什么“预测出来了”还远远不够}

前面的章节已经让我们看到，模型可以在小型教学数据上给出相当不错的分类或回归结果。但只要进入真实科学场景，一个更难的问题马上出现：\textbf{这个结果到底可信到什么程度？}

可信度至少涉及三层不同的问题：

\begin{itemize}
    \item \textbf{不确定度}：如果重新抽样或重新观测，结果会波动多大；
    \item \textbf{解释性}：模型主要依赖了哪些特征、在哪些样本上这样做；
    \item \textbf{失效边界}：样本是否已经跑到训练数据覆盖范围之外。
\end{itemize}

如果这三层没有分开看，你很容易把一个表面稳定的预测当成可靠知识。

\section{一个教学版光度红移可信度问题}

本章配套 notebook 使用 \nolinkurl{photoz_trust_demo.csv}。这个小数据集包含：

    \item 四个测光颜色特征：\texttt{u\_g}、\texttt{g\_r}、\texttt{r\_i}、\texttt{i\_z}；
    \item 一个参考光谱红移 \texttt{specz}；
    \item 三种角色：训练集、普通测试集、边界外样本。

\begin{itemize}
    \item 域内样本上的正常误差；
    \item bootstrap 区间的宽度；
    \item 边界外样本虽然也能被预测，但误差会显著变大。
\end{itemize}

\section{不确定度不等于一个数字，而是一类问题}

很多人第一次接触模型不确定度时，会希望得到一个统一的“误差条”。但在科学建模里，不确定度来源至少可以分成几类：

\begin{itemize}
    \item 训练样本有限带来的统计波动；
    \item 观测误差本身的传播；
    \item 模型形式不完备带来的系统偏差；
    \item 样本跑出训练分布时的额外失效风险。
\end{itemize}

这意味着：\textbf{任何单一不确定度估计，都只是在覆盖其中某一部分。}

\section{Bootstrap：回答“如果训练集稍有变化，结果会怎样”}

\textbf{Bootstrap} 的核心做法，是从训练集反复有放回重采样，每次都重新训练模型，再观察预测结果如何波动。

\begin{examplebox}[bootstrap 的直觉]
\begin{lstlisting}[style=pythonstyle]
for repeat in range(B):
    resampled_train = sample_with_replacement(train)
    fit model
    predict target
\end{lstlisting}
\end{examplebox}

它能帮助你估计：

\begin{itemize}
    \item 模型对训练样本扰动是否敏感；
    \item 某个预测值附近大致有多少统计波动；
    \item 哪些点的区间较窄、哪些点的区间较宽。
\end{itemize}

但 bootstrap 也有明确边界：\textbf{它主要在回答“在训练分布附近重采样会怎样”，并不自动知道你是否已经跑出训练分布。}

\section{预测区间不是万能保证}

在 notebook 里，你会看到普通测试样本的 bootstrap 区间比较窄，而且预测误差也较小。但对边界外样本，即使 bootstrap 区间看起来并不离谱，模型仍然可能出现明显偏差。

这提醒我们：一个窄区间并不一定说明模型“靠谱”，它也可能只是说明：

\begin{itemize}
    \item 所有 bootstrap 模型都在同一个方向上做了相似的外推；
    \item 模型在分布外区域很自信，但自信错了。
\end{itemize}

所以，\textbf{预测区间描述的是波动，不等于全面可信度。}

\section{特征重要性：模型主要依赖了什么}

很多时候，我们想知道模型到底最看重哪些输入特征。一个常见而直接的起点是\textbf{permutation importance}：

\begin{itemize}
    \item 先计算正常情况下的误差；
    \item 再把某个特征打乱；
    \item 观察误差恶化了多少。
\end{itemize}

如果打乱某个特征后误差明显上升，说明模型较依赖这个特征。

这个指标很有用，但也必须小心理解：

\begin{itemize}
    \item 它反映的是\textbf{模型依赖}，不是物理因果；
    \item 高相关特征可能互相替代，导致重要性被分散或低估；
    \item 结果依赖于你选的模型、数据切分和评价指标。
\end{itemize}

\section{局部解释：为什么这个样本会得到这个预测}

全局重要性告诉你“整体上模型看重什么”，但很多科学场景还会问：\textbf{为什么这个具体样本得到这个预测？}

这时我们需要局部解释。SHAP 一类方法的核心思想，是把单个预测拆成若干特征贡献，并与一个基线值比较。它的吸引力在于：

\begin{itemize}
    \item 既能做单样本解释；
    \item 又能把解释结果汇总成整体模式；
    \item 形式上很适合可视化与沟通。
\end{itemize}

但在本科阶段，最重要的不是马上掌握完整的 Shapley 计算，而是先建立正确态度：\textbf{解释方法是在描述模型行为，不是在直接证明物理机制。}

\section{解释性不是因果性}

这是整章里最需要牢牢守住的一条边界。即使某个特征在 permutation importance 里很高，或者在局部解释里对某个预测贡献很大，也不代表：

\begin{itemize}
    \item 这个特征在物理上就是根本原因；
    \item 去改变这个特征就一定会按同样方式改变结果；
    \item 模型学到的是物理机制，而不是训练样本中的统计相关。
\end{itemize}

因此，解释性工具的角色，更接近“帮助你审计模型在利用什么信息”，而不是“直接提供自然规律证明”。

\section{模型失效边界：最容易被忽略的一层}

如果一个样本在特征空间里离训练集非常远，那么无论你的误差条看起来多漂亮、解释图做得多细致，模型都可能已经进入失效区域。

在 notebook 里，我们会计算每个样本到训练集最近邻的标准化距离。你会发现：

\begin{itemize}
    \item 域内测试样本距离较小；
    \item 边界外样本距离非常大；
    \item 恰恰是这些距离很大的点，回归误差明显恶化。
\end{itemize}

这一步的意义非常大，因为它告诉我们：\textbf{可信度不是只看模型输出本身，还要看样本是否仍在模型“见过的世界”里。}

\section{覆盖范围检查的一种最小算法}

覆盖范围检查可以从一个很朴素的距离思想开始。先用训练集定义标准化尺度，再计算待预测样本到训练集中最近邻的距离：

\[
d_{\min}(x)=\min_{x_j\in \mathcal{D}_{\mathrm{train}}}
\left\|z(x)-z(x_j)\right\|,
\]

其中 $z(\cdot)$ 表示使用训练集均值和标准差得到的标准化特征。若 $d_{\min}$ 很大，说明这个样本在训练集中没有相近邻居，模型预测就应降级为高风险。

\begin{examplebox}[覆盖范围检查骨架]
\begin{lstlisting}[style=pythonstyle]
scaler = fit_scaler(train_rows)
train_z = transform(scaler, train_rows)
target_z = transform(scaler, target_rows)

nearest_distance = distance_to_nearest_training_row(target_z, train_z)
coverage_flag = nearest_distance < coverage_threshold
\end{lstlisting}
\end{examplebox}

这个方法并不完美，但它足够建立一个关键习惯：任何模型预测都应同时报告“这个样本离训练经验有多远”。离得越远，数值预测越应该被谨慎使用。

\section{选择效应和观测条件也会伪装成“规律”}

真实科学数据里，模型失败不仅来自分布外样本，还来自更隐蔽的选择效应。例如：

\begin{itemize}
    \item 高红移样本更暗，导致测光误差系统增大；
    \item 训练集主要覆盖亮源，而测试时却去预测暗源；
    \item 某些类型只在特定巡天、特定深度或特定信噪比下出现。
\end{itemize}

如果这些效应没有被记录下来，模型就可能把观测条件学成“物理规律”。这也是为什么可信度评估不能只停留在模型分数和解释图上。

\section{可信的机器学习结果，至少要同时过三道检查}

\begin{enumerate}
    \item \textbf{误差检查}：训练、验证、测试上的误差是否合理；
    \item \textbf{解释检查}：模型依赖的特征是否符合你对问题的基本理解；
    \item \textbf{覆盖检查}：待预测样本是否仍处在训练分布附近。
\end{enumerate}

只有三者一起过关，模型输出才有资格进入科学解释。

\section{Trust Statement：一个最小可信度声明模板}

当模型结果要进入论文草稿、项目报告或课程展示时，最好附上一段简短但明确的可信度声明。它的目的不是显得保守，而是告诉读者：你知道这个模型在哪些地方可靠、哪些地方不能过度解释。

\begin{enumerate}
    \item \textbf{误差声明}：报告测试误差、典型残差或混淆模式，而不是只给单个预测例子；
    \item \textbf{不确定度声明}：说明 bootstrap、交叉验证或重复切分覆盖的是哪类波动；
    \item \textbf{解释性声明}：说明特征重要性和局部解释描述的是模型行为，不是物理因果；
    \item \textbf{适用范围声明}：说明训练集覆盖的颜色、亮度、红移、信噪比或其他关键范围；
    \item \textbf{失效风险声明}：指出分布外样本、选择效应和观测条件变化可能如何破坏结论。
\end{enumerate}

\begin{examplebox}[可信度声明的最小骨架]
\begin{lstlisting}[style=pythonstyle]
trust_statement = {
    "model_output": predicted_photoz,
    "main_metric_or_diagnostic": test_error_summary,
    "distribution_status": "in_distribution / out_of_distribution / unclear",
    "main_uncertainty_source": "finite training sample",
    "uncertainty_method": "bootstrap over training rows",
    "main_feature_dependence": top_features,
    "interpretability_scope": "model behavior, not causal proof",
    "known_failure_boundary": coverage_summary,
    "human_review_needed": review_flag,
    "supported_claim": supported_claim,
    "unsupported_claim": unsupported_claim,
}
\end{lstlisting}
\end{examplebox}

如果一个模型在训练覆盖范围内表现稳定，但对边界外样本误差明显升高，那么 Trust Statement 中就应明确写出这一点。科学可信度不是把模型包装得更强，而是把模型能承担的证据责任说清楚。

在最终项目里，这段 Trust Statement 会把 Dataset Contract、Model Experiment Record、evaluation artifact 和 error analysis artifact 连接起来：模型怎样训练、误差怎样估计、解释能说到哪里、失效边界在哪里，都必须同时出现。少掉 uncertainty、interpretability 或 failure boundary 中任意一项，可信度声明就会退化成一句空泛的“模型表现良好”。

\section{从预测表到可信度表}

一个可审查的模型输出表，不应只包含预测值。至少应包含：

\begin{itemize}
    \item 样本编号和输入数据版本；
    \item 预测值或预测类别；
    \item 误差、区间或重复实验波动；
    \item 最近训练邻居距离或覆盖范围标记；
    \item 主要解释特征或局部解释摘要；
    \item 是否建议人工复核。
\end{itemize}

\begin{examplebox}[可信度表字段]
\begin{lstlisting}[style=pythonstyle]
prediction_audit_row = {
    "object_id": object_id,
    "prediction": predicted_photoz,
    "bootstrap_interval": (low, high),
    "nearest_train_distance": nearest_distance,
    "coverage_flag": coverage_flag,
    "top_explanation_features": top_features,
    "review_recommendation": "manual_review_if_out_of_coverage",
}
\end{lstlisting}
\end{examplebox}

这类表格会显著改变模型结果的使用方式。读者不再只看到一个预测值，而是同时看到这个预测值来自哪里、波动多大、是否外推、模型依赖了哪些信息，以及是否需要人工降级处理。

\begin{warnbox}[可信度评估中的常见误区]
\begin{itemize}
    \item 把 bootstrap 区间当作“已经包含全部不确定度”；
    \item 把特征重要性直接解释成因果关系；
    \item 看到解释图很漂亮，就忽略样本是否已经分布外；
    \item 只报告模型分数，不报告覆盖范围与选择效应；
    \item 对边界外样本给出很具体的数值预测，却不提醒用户风险。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这一章，AI 助手适合帮助你：

\begin{itemize}
    \item 整理 bootstrap 结果与预测区间表格；
    \item 检查 permutation importance 或局部解释代码是否与定义一致；
    \item 帮你草拟“模型适用范围”与“结果限制”说明文字；
    \item 协助你把“模型解释”和“科学解释”分成两层清楚记录。
\end{itemize}

但哪些误差来源尚未覆盖、某个解释是否可能只是选择效应、以及这份模型结果能不能写进科学结论，仍然必须由你自己判断。

\end{exercisebox}

\section{本章小结}

不确定度、解释性和可信度不是模型训练后的附属装饰，而是决定结果能否真正进入科学论证的核心环节。bootstrap 告诉你结果会如何波动，特征重要性和局部解释告诉你模型如何使用输入，而覆盖范围和选择效应则提醒你：再稳定、再可解释的输出，也可能在分布外区域失效。真正可信的科学机器学习，不是“有预测值就结束”，而是把误差、解释和适用边界一起放到台面上。
