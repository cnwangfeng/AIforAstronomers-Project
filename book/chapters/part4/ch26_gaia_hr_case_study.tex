\chapter{Gaia HR 图与恒星分类案例}

\section{案例导读：把物理图像变成模型工作流}

这是 Part IV 的第一个案例，故意选择 HR 图作为入口。原因很简单：HR 图本身已经有清楚的天体物理结构，机器学习在这里不是用来“制造发现”，而是帮助我们把结构识别、样本筛选、类别判断和错分复查组织成一条可复查的工作流。

这个案例最适合训练一个关键转变：从“我会画 HR 图”走向“我能围绕 HR 图交付一个小型项目包”。项目包里既要有 Part II 的数据契约、单位和派生量，也要有 Part III 的 baseline、模型记录、错误分析和解释边界。模型只是证据链中的一环，而不是案例的全部。

读这一章时，请把注意力放在错分样本和边界区域。恒星分类最有教学价值的地方，往往不是主序星、巨星和白矮星可以被分开，而是哪些点落在边界上，为什么这些边界同时带着数据质量、物理演化和标签定义的问题。

\section{案例目标}

这一章是本书进入 Part IV 后的第一个完整案例。目标不是再介绍一个新算法，而是把前面 Part II 和 Part III 的方法重新装配成一条更接近真实项目的工作流：

\begin{enumerate}
    \item 从 Gaia 风格表格数据出发；
    \item 做最基本的数据质量筛选；
    \item 计算绝对星等并构造 HR 图坐标；
    \item 用无监督方法观察结构；
    \item 用一个简单监督模型完成恒星类别分类；
    \item 最后回到天体物理解释本身。
\end{enumerate}

\section{为什么 HR 图是一个绝佳案例}

HR 图几乎是恒星天体物理里最经典的二维结构之一。只要把颜色放在横轴、绝对星等放在纵轴，很多不同类型恒星就会自然占据不同区域：

\begin{itemize}
    \item 主序星大致形成一条从蓝亮到红暗的带状结构；
    \item 红巨星在红且亮的区域形成分支；
    \item 白矮星通常更蓝、更暗。
\end{itemize}

这使它非常适合用来演示：

\begin{itemize}
    \item 数据清洗为什么重要；
    \item 无监督学习如何帮助你识别结构；
    \item 监督学习如何在已知类别上做自动判别；
    \item 机器学习结果如何返回到物理解释。
\end{itemize}

更重要的是，这个案例不是“为了用算法而找数据”，而是一个原本就有强物理结构的问题。正因为结构先存在，算法才有意义。

\section{案例问题：我们到底想完成什么}

本案例的核心问题并不是“哪种分类器分数最高”，而是：

\begin{itemize}
    \item 数据经过最基本质量筛选后，HR 图上的恒星结构是否清楚；
    \item 不看标签时，这些结构会不会自然浮现；
    \item 如果用最简单的监督方法做自动判别，错分会发生在哪里；
    \item 这些错分究竟是算法问题，还是物理边界本来就模糊。
\end{itemize}

这条问题链非常适合作为本科生案例训练，因为它把“数据处理、建模、解释”放在了同一张图里。

\section{数据与质量筛选}

配套 notebook 使用 \nolinkurl{gaia_hr_case_demo.csv}。它是一个小型 Gaia 风格教学数据集，包含：

\begin{itemize}
    \item 颜色指标 \texttt{bp\_rp}；
    \item 视差 \texttt{parallax\_mas}；
    \item 视差误差 \texttt{parallax\_error\_mas}；
    \item 表观星等 \texttt{g\_mag}；
    \item 教学参考类别 \texttt{stellar\_class}。
\end{itemize}

这里故意保留了两个低质量样本，它们的视差信噪比较低。这样做是为了让学生明确看到：\textbf{不是所有表格里的行都应该直接拿来画 HR 图。}

本案例首先使用一个最基本的质量标准：

\[
\texttt{parallax\_over\_error}
=
\frac{\texttt{parallax\_mas}}
{\texttt{parallax\_error\_mas}}.
\]

随后保留

\[
\texttt{parallax\_over\_error} > 5
\]

的样本。这个阈值当然不是唯一正确选择，但它足以建立一个重要习惯：先想清楚数据质量，再谈机器学习。

\section{从视差和表观星等到绝对星等}

一旦有了视差，我们就可以把表观星等转成绝对星等。对本案例来说，最方便的写法是：

\[
M_G = g_{\mathrm{mag}} + 5\log_{10}(p_{\mathrm{mas}}) - 10.
\]

\begin{examplebox}[绝对星等公式]
\begin{lstlisting}[style=pythonstyle]
M_G = g_mag + 5 * log10(parallax_mas) - 10
\end{lstlisting}
\end{examplebox}

有了 \texttt{bp\_rp} 和 \texttt{M\_G}，我们就得到绘制 HR 图所需的两个核心坐标。

\section{第一步：先看图，再上模型}

在很多项目里，最容易犯的错误就是急着训练模型，却没有先检查数据在物理空间里长什么样。本案例刻意把“先看 HR 图”放在前面，因为这一步能立刻帮助你判断：

\begin{itemize}
    \item 主序星、巨星和白矮星是否确实占据不同区域；
    \item 低质量样本是否会污染结构；
    \item 颜色和绝对星等是否已经足以分开这些类别。
\end{itemize}

这一步看上去朴素，但它常常决定后面模型工作是否有意义。因为一旦图上结构本身就混乱，你更该回去检查数据、误差和样本定义，而不是盲目堆更复杂的模型。

\section{第二步：先让无监督方法看结构}

接下来，notebook 会把清洗后的样本送入一个小型的 \texttt{k-means} 聚类。之所以先做无监督步骤，是因为我们想回答：

\begin{itemize}
    \item 即使暂时不看标签，主序星、巨星和白矮星会不会自然形成不同群；
    \item 算法看到的结构，是否与天体物理直觉一致；
    \item 那些不太合群的样本，是否值得单独回头看。
\end{itemize}

在这个教学数据里，答案是相当清楚的：三类对象在 HR 图上本来就高度分离，因此聚类结果会与参考类别非常接近。

但这里必须保持清醒：这不是在证明“宇宙一定只分成三个离散簇”，而是在说明当前数据、当前特征下，\textbf{结构分离得足够清楚。}

\section{第三步：再做一个简单监督分类器}

在看过无监督结构之后，我们再回到监督学习，使用一个非常朴素的最近中心分类器。这样设计是有意为之：

\begin{itemize}
    \item 我们不需要复杂模型也能完成这个案例；
    \item 更简单的模型更有利于把注意力放在科学结构上；
    \item 如果简单模型已经表现很好，就没有必要为了“更先进”而无条件堆复杂度。
\end{itemize}

在 notebook 里，我们用 \texttt{bp\_rp} 和 \texttt{M\_G} 两个特征做分类，并在一个固定测试集上报告准确率与混淆矩阵。

\section{错分最值得看什么}

在这个案例里，错分本身比准确率更有教学价值。因为每一个错分都可能对应：

\begin{itemize}
    \item 某个样本靠近两类边界；
    \item 质量筛选阈值仍不够严格；
    \item 教学标签本身就是某种简化；
    \item 模型确实忽略了更细的物理信息。
\end{itemize}

也就是说，错分不是“成绩单上的坏消息”，而是帮助你理解结构边界的入口。

\section{为什么这个案例特别适合教学}

这个案例的好处在于，它把很多教材里分散讲的内容揉在了一起：

\begin{itemize}
    \item Part II 的数据处理：视差、绝对星等、HR 图；
    \item Part III 的无监督学习：结构探索与聚类；
    \item Part III 的监督学习：简单分类与评估；
    \item 科学解释：主序、巨星、白矮星在 HR 图上的物理位置。
\end{itemize}

换句话说，这一章不是“又一个 notebook”，而是第一次让学生真正体验到：\textbf{一个天文数据项目是怎样从原始表格一路走到物理结论的。}

\section{这个案例的边界在哪里}

当然，这个案例仍然是教学版，而不是完整科研项目。它省略了很多现实中的复杂性，例如：

\begin{itemize}
    \item 更复杂的测光误差与消光修正；
    \item 双星、变星或亚结构带来的模糊边界；
    \item 更真实的 Gaia 选择效应；
    \item 更大样本下的类不平衡和标签不完整问题。
\end{itemize}

因此，我们应该把它理解成一个\textbf{结构清楚的练习场}：先把方法和解释链条走顺，再准备进入更复杂的数据。

\section{一个最小 Gaia HR 图案例报告模板}

这个案例的报告不应只写“分类准确率是多少”。真正需要展示的是：数据筛选、HR 图结构、算法结果和物理解释是否连成一条证据链。一个最小报告可以按下面顺序组织：

\begin{enumerate}
    \item \textbf{问题定义}：说明你想用 Gaia 风格数据识别哪些恒星结构，而不是单纯追求分类分数；
    \item \textbf{数据与质量筛选}：列出使用字段、视差信噪比阈值、被剔除样本及其原因；
    \item \textbf{HR 图构造}：写明绝对星等公式，并解释颜色--星等坐标为什么有物理意义；
    \item \textbf{结构探索}：报告无监督聚类是否与主序、巨星、白矮星区域一致；
    \item \textbf{监督分类与错分复查}：报告测试结果、混淆矩阵，以及最值得解释的边界样本；
    \item \textbf{适用边界}：说明教学样本无法覆盖真实 Gaia 数据中的消光、双星、变星和选择效应。
\end{enumerate}

\begin{examplebox}[Gaia HR 图案例报告骨架]
\begin{lstlisting}[style=pythonstyle]
gaia_hr_report = {
    "quality_cut": "parallax_over_error > 5",
    "hr_axes": ["bp_rp", "M_G"],
    "unsupervised_summary": cluster_summary,
    "classification_summary": confusion_matrix,
    "boundary_examples": misclassified_or_low_margin_ids,
    "physical_claim": "teaching-scale HR structure is recovered",
    "limits": ["small sample", "no extinction correction", "simplified labels"],
}
\end{lstlisting}
\end{examplebox}

写这样的报告时，最重要的不是把每个数字都列出来，而是让读者看见：你为什么相信这些结构是真的、为什么这些错分值得讨论、以及为什么当前结论还不能直接外推到完整巡天。

在 Part IV case package 中，这份报告应拆成同一个 Model Experiment Packet：HR 图坐标、视差质量和类别来源进入 Dataset Contract；聚类、最近中心分类、混淆矩阵和边界样本复查进入 Model Experiment Record、Evaluation artifact 与 Error analysis artifact；消光、双星、变星、选择效应和简化标签则进入 Trust Statement。进入 Part VI 时，学生可以直接扩展这套证据，而不是重新发明项目结构。

本章配套 notebook 的最后一步会导出一份最小样板包：

\begin{examplebox}[Ch26 最小 case package]
\begin{lstlisting}[style=terminal]
results/ch26_gaia_hr_case/
  dataset_contract_gaia_hr.md
  model_experiment_record_gaia_hr.md
  trust_statement_gaia_hr.md
  confusion_matrix.csv
  boundary_examples.csv
  figures/
    hr_diagram_boundary_examples.png
\end{lstlisting}
\end{examplebox}

这份清单的作用，是让 Gaia HR 图案例成为“物理结构先于模型”的可仿写样板：先固定数据质量和 HR 图坐标，再记录聚类、分类、边界样本和不能外推的科学限制。

\begin{warnbox}[这个案例中的常见误区]
\begin{itemize}
    \item 没有先做视差质量筛选，就直接画 HR 图；
    \item 把教学数据上的高准确率误解成真实巡天数据同样容易；
    \item 只报告模型分类结果，不回到 HR 图空间看错分发生在哪里；
    \item 忽视样本量很小这一点，把当前结论外推得太远；
    \item 忘记颜色和绝对星等本身已经承载了很强的物理结构。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这个案例里，AI 助手适合帮助你：

\begin{itemize}
    \item 检查绝对星等计算与质量筛选代码；
    \item 帮你整理聚类结果、分类结果和物理解释之间的对应关系；
    \item 帮你生成一个简洁的案例报告框架；
    \item 协助你把“模型结果”与“HR 图上的物理结构”写成同一条清楚叙事。
\end{itemize}

但样本为什么会分成这些区域、错分是否具有物理意义、以及这个案例能否外推到更真实的 Gaia 数据，仍然需要你自己判断。

\section{练习}

\begin{exercisebox}[基础练习]
在配套 notebook 中分别使用 \texttt{parallax\_over\_error > 3} 和 \texttt{parallax\_over\_error > 10} 重新筛选样本，比较 HR 图上主序、巨星和白矮星区域的清晰程度。记录被移除或重新保留的对象，并说明阈值改变对物理解释有什么影响。
\end{exercisebox}

\begin{exercisebox}[进阶练习]
选择一个被分类器错分或最接近类别边界的样本，回到它的 \texttt{bp\_rp}、\texttt{M\_G}、视差信噪比和参考类别。请写一段错误案例分析，判断它更像是数据质量问题、类别边界模糊，还是模型表示过于简单。
\end{exercisebox}

\begin{exercisebox}[开放练习]
假设你要把本章教学版 workflow 扩展到真实 Gaia 样本。请写一页扩展方案，至少包含：消光修正、视差零点、双星或变星污染、样本选择效应、类别标签来源，以及你会怎样保留一个简单 baseline 作为参照。
\end{exercisebox}

\section{本章小结}

Gaia HR 图案例展示了一个最重要的事实：好的天文机器学习项目，并不是从“最复杂模型”开始，而是从可靠数据、清楚结构和可解释特征开始。只要颜色与绝对星等已经把主序、巨星和白矮星分开，简单的聚类和分类方法就足以建立一个完整而可信的工作流。案例真正的价值，不在于分数多高，而在于它让数据处理、结构探索、监督建模和物理解释第一次完整地接到了一起。
