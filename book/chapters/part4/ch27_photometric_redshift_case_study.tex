\chapter{光度红移估计案例}

\section{案例目标}

这一章把前面 Part III 的回归、模型评估和可信度分析组合成一个更接近真实巡天任务的案例。我们的目标是：

\begin{enumerate}
    \item 从一个小型 SDSS 风格测光表出发；
    \item 建立一个故意朴素的一维回归 baseline；
    \item 再用多特征线性回归改进预测；
    \item 用残差分析检查模型在不同类型星系上的表现；
    \item 明确模型能做什么、不能做什么，以及为什么不能过度外推。
\end{enumerate}

\section{问题背景：为什么光度红移这么重要}

如果每个星系都能拿到高质量光谱，那么红移测量会直接得多。但现实里，大样本巡天通常只能为少量天体获得光谱，其余对象往往只有多波段测光。因此，天文学里长期存在一个非常实用的问题：

\begin{quote}
能不能根据少量测光颜色，快速估计星系红移？
\end{quote}

这就是光度红移问题。它既是一个典型回归任务，也是一个非常适合教学的科学机器学习案例，因为它天然包含：

\begin{itemize}
    \item 特征选择；
    \item 训练集和测试集划分；
    \item baseline 与改进模型的比较；
    \item 残差分析；
    \item 训练样本覆盖范围的限制。
\end{itemize}

更重要的是，光度红移并不是一个“只看误差数字”的工程任务。即使整体误差不高，模型也可能在某些颜色区域、某些星系类型或某些稀有对象上系统失效。

\section{数据：一个教学版 SDSS 风格小样本}

配套 notebook 使用 \nolinkurl{photoz_case_demo.csv}。数据表包含：

\begin{itemize}
    \item 颜色特征 \texttt{u\_g}, \texttt{g\_r}, \texttt{r\_i}, \texttt{i\_z}；
    \item 一个亮度特征 \texttt{r\_mag}；
    \item 教学用类型标签 \texttt{galaxy\_type}；
    \item 参考红移 \texttt{specz}。
\end{itemize}

这里的 \texttt{galaxy\_type} 并不是给模型直接使用的，而是为了帮助我们在误差分析阶段回答一个更科学的问题：\textbf{模型在不同类型星系上是否同样可靠？}

这个设置非常重要，因为它把“回归分数”和“科学对象差异”连接在了一起。模型并不是在真空中对一堆抽象点做拟合，而是在面对不同物理族群。

\section{案例真正想回答的问题}

表面上看，我们是在做红移回归；但从教学角度，这一章真正关心的是下面几件事：

\begin{itemize}
    \item 一个单色 baseline 到底能做到什么程度；
    \item 多特征模型的改进是普遍发生，还是只在某些对象上发生；
    \item 整体误差下降之后，是否还存在局部上很危险的失效样本；
    \item 训练集没有覆盖好的对象类型，会怎样在测试集里暴露出来。
\end{itemize}

也就是说，本案例不只是“做一个回归模型”，而是在训练学生如何读懂一个回归模型的边界。

\section{第一步：先做一个故意朴素的 baseline}

案例先从只用 \texttt{g-r} 单一颜色预测红移开始。这样做有两个好处：

\begin{itemize}
    \item 它足够简单，能够帮助学生直观看到“颜色随红移变化”的趋势；
    \item 它也足够弱，便于和后面的多特征模型作比较。
\end{itemize}

在 notebook 里，这个 baseline 使用一元线性回归：

\[
z_{\mathrm{pred}} = \beta_0 + \beta_1 (g-r).
\]

\begin{examplebox}[一元 baseline]
\begin{lstlisting}[style=pythonstyle]
z_pred = intercept + slope * (g_r)
\end{lstlisting}
\end{examplebox}

这一步的教学意义，不在于它一定准确，而在于它给我们提供了一个最低对照组。后面的任何改进，都应该拿它来比较。

\section{为什么这个 baseline 很有价值}

一个常见误区是把 baseline 理解成“只是在走流程”。实际上，在光度红移问题里，这个单色 baseline 非常有价值，因为它强迫我们先问：

\begin{itemize}
    \item 单一颜色到底承载了多少红移信息；
    \item 这个关系是近似线性，还是强烈弯曲；
    \item 某些星系类型是否会天然偏离这种简单趋势。
\end{itemize}

只有先把这些最基本的问题想清楚，我们才有资格说多特征模型到底带来了什么。

\section{第二步：再上一个多特征回归模型}

接下来，我们把四个颜色指标和 \texttt{r\_mag} 一起放入模型。对教学案例来说，多特征线性回归已经足以展示一条完整而清楚的思路：

\begin{itemize}
    \item 多个测光量共同提供红移信息；
    \item 模型结构仍然可解释；
    \item 可以直接比较和 baseline 相比误差是否下降。
\end{itemize}

notebook 中使用的是纯 Python 正规方程求解的最小实现。模型形式可以写成：

\[
z_{\mathrm{pred}} =
\beta_0
+ \beta_1 (u-g)
+ \beta_2 (g-r)
+ \beta_3 (r-i)
+ \beta_4 (i-z)
+ \beta_5 r_{\mathrm{mag}}.
\]

\begin{examplebox}[多特征线性模型]
\begin{lstlisting}[style=pythonstyle]
z_pred = beta0
       + beta1 * (u_g)
       + beta2 * (g_r)
       + beta3 * (r_i)
       + beta4 * (i_z)
       + beta5 * (r_mag)
\end{lstlisting}
\end{examplebox}

这并不是在说线性模型总是最好，而是在强调：\textbf{进入更复杂模型之前，你应先建立一个清楚、可复现、可解释的中等复杂度基线。}

\section{第三步：看总体指标，更要看残差分布}

光度红移任务里，只报告一个平均误差数字远远不够。你至少还应问：

\begin{itemize}
    \item 哪些对象误差最大；
    \item 残差是否有系统性偏差；
    \item 某些类型星系是否明显更难预测。
\end{itemize}

因此，本案例不仅计算 MAE、RMSE 和平均偏差，也会把测试集里的单个对象列出来，逐个检查预测残差。

残差定义为：

\[
r = z_{\mathrm{pred}} - z_{\mathrm{spec}}.
\]

\begin{examplebox}[残差定义]
\begin{lstlisting}[style=pythonstyle]
residual = z_pred - specz
\end{lstlisting}
\end{examplebox}

如果平均误差看起来不错，但少数样本残差始终偏大，那么这个模型在科学使用上仍然可能有明显风险。

\section{第四步：把误差放回物理对象类型}

这是本案例最重要的一步之一。你会看到：在这个教学数据里，多特征线性回归比单色 baseline 好很多，但它并没有“解决一切问题”。

特别是对少见类型，例如 \texttt{blue\_compact} 星系，如果训练集中只有极少例子，那么模型即使在整体指标上表现不错，也可能在这类对象上明显失真。换句话说，\textbf{总体平均表现和具体对象表现，不是一回事。}

这一步是案例章与方法章最大的区别：方法章告诉你如何建模，案例章要求你把模型误差重新放回天体对象本身。

\section{为什么少见类型更容易出问题}

对于少见类型，模型失败的原因通常不是单一的。常见可能性包括：

\begin{itemize}
    \item 训练集中同类样本太少，模型没有足够经验；
    \item 该类型的颜色--红移关系本身更复杂；
    \item 某些亮度或颜色区间正好处在训练分布边缘；
    \item 少见对象更容易受选择效应和测光误差影响。
\end{itemize}

因此，案例中最值得警惕的并不是“误差大”这件事本身，而是\textbf{误差是否在某一类对象上系统聚集。}

\section{为什么这个案例很像真实项目}

和前面 Part III 的方法章相比，这一章更接近一个完整的小项目，因为它同时包含：

\begin{itemize}
    \item 巡天风格表格数据；
    \item baseline 与改进模型的对照；
    \item 测试集残差表；
    \item 按对象类型检查模型失效边界；
    \item 对“训练样本覆盖不足”这一科学问题的直接讨论。
\end{itemize}

这也是为什么我们把它放进 Part IV：重点不再只是“会不会调用模型”，而是能否把模型结果重新放回科学上下文中解释。

\section{如果有标准库，还可以怎么扩展}

notebook 最后还提供了一个可选的 \texttt{scikit-learn} 随机森林对照块。如果环境里装有相关库，你可以进一步比较：

\begin{itemize}
    \item 线性模型和树模型谁在这个数据上更稳；
    \item 非线性模型是否更能处理少数复杂对象；
    \item 当样本很小时，更复杂模型是否真的值得引入。
\end{itemize}

但请记住：复杂模型不是默认答案。只有当它带来稳定、可验证的收益时，复杂度才是值得的。

\section{这个案例的边界在哪里}

尽管这个案例已经比方法章更像一个完整小项目，但它仍然是教学版而不是正式巡天流水线。它省略了很多真实光度红移任务中的复杂性，例如：

\begin{itemize}
    \item 消光修正与模板退化；
    \item 更复杂的测光误差传播；
    \item 光谱训练样本的选择偏差；
    \item 更深巡天中高红移样本稀缺带来的分布外风险；
    \item 不同仪器和数据版本之间的系统差异。
\end{itemize}

因此，我们不能把教学数据上的顺利结果，直接等价于真实巡天中的可发布模型。

\section{一个最小光度红移案例报告模板}

光度红移报告的核心，不是证明模型“会预测”，而是说明它在什么对象、什么颜色范围、什么训练覆盖条件下可以被信任。一个最小报告可以包含：

\begin{enumerate}
    \item \textbf{任务与标签}：说明输入是测光颜色和亮度，目标是参考光谱红移；
    \item \textbf{baseline 对照}：报告单色 baseline 的误差，作为最低参照；
    \item \textbf{改进模型}：报告多特征模型的 MAE、RMSE、偏差和相对改进；
    \item \textbf{残差诊断}：列出最大残差样本，并检查它们是否集中在某类星系或某个颜色范围；
    \item \textbf{覆盖与外推}：说明训练样本是否覆盖测试对象，哪些对象可能处在分布边缘；
    \item \textbf{结论强度}：明确哪些说法只适用于教学数据，哪些工作需要真实巡天数据继续验证。
\end{enumerate}

\begin{examplebox}[光度红移案例报告骨架]
\begin{lstlisting}[style=pythonstyle]
photoz_report = {
    "baseline": {"features": ["g_r"], "rmse": baseline_rmse},
    "improved_model": {
        "features": ["u_g", "g_r", "r_i", "i_z", "r_mag"],
        "rmse": model_rmse,
        "bias": mean_residual,
    },
    "largest_residuals": residual_table[:5],
    "type_dependent_failure": type_error_summary,
    "coverage_warning": coverage_notes,
}
\end{lstlisting}
\end{examplebox}

如果多特征模型平均误差下降，但最大残差仍然来自少见类型，那么报告中应同时写出“整体改进”和“局部失效”。这不是削弱结果，而是在把模型放回科学使用场景里。

在 capstone 包里，这份报告会让 Baseline card、Model card 和 Evaluation card 相互对齐：baseline 提供最低参照，多特征模型说明改进来源，残差诊断和覆盖警告说明哪些对象仍然不能被自动信任。

\begin{warnbox}[光度红移案例中的常见误区]
\begin{itemize}
    \item 只看整体 MAE，不检查最大残差来自哪些对象；
    \item 训练样本中几乎没有某类星系，却仍然对该类预测过度自信；
    \item 把教学数据上的表现直接外推到真实 SDSS 或更深巡天；
    \item 忽视测光系统差异、消光、样本选择效应等现实复杂性；
    \item 把经验预测模型误当成物理机制本身。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这个案例里，AI 助手适合帮助你：

\begin{itemize}
    \item 检查训练/测试划分和残差统计代码；
    \item 帮你把 baseline 和改进模型的结果整理成表格；
    \item 草拟“误差来源与模型边界”的说明；
    \item 协助你把“对象类型差异”和“模型失效边界”联系起来写清楚。
\end{itemize}

但哪个对象属于训练分布外、哪种残差模式值得科学上继续追踪、以及模型是否能用于真实巡天数据，仍然需要你自己判断。

\section{练习}

\begin{exercisebox}[基础练习]
用本章 notebook 的结果，找出测试集中绝对残差最大的几个对象，记录它们的颜色、真实红移、预测红移和对象类型，并用一段话说明这些失败是否集中在某类对象上。
\end{exercisebox}

\begin{exercisebox}[进阶练习]
改变一次训练/测试划分的随机种子，重新比较 baseline 与改进模型的 MAE。观察两者差距是否稳定，并说明如果差距不稳定，应该如何在报告中降低结论强度。
\end{exercisebox}

\begin{exercisebox}[开放练习]
假设你要把这个教学版光度红移 workflow 扩展到真实巡天数据。请写出至少五项必须新增的数据检查或质量控制步骤，例如测光误差、消光修正、训练样本选择偏差、红移范围覆盖或仪器系统差异。
\end{exercisebox}

\section{本章小结}

光度红移案例展示了一个很关键的事实：一个看上去“整体不错”的回归模型，仍然可能在少见对象上失败。真正可靠的科学机器学习，不只是提高平均精度，还包括识别模型在哪些类型、哪些范围、哪些样本条件下不再可信。本章的价值，不在于把误差压到最低，而在于让你第一次系统地把 baseline、改进模型、残差分析和对象类型解释放进同一条案例叙事里。
