\chapter{星系形态分类案例}

\section{案例导读：图像分类先从可解释 baseline 开始}

星系形态分类看起来天然属于深度学习：输入是图像，输出是形态类别。但这正是本章要刻意放慢的地方。一个健康的图像分类项目，不能一开始就跳到卷积网络，而要先弄清楚图像是否可比、标签是否稳定、质量筛选是否合理、传统形态特征能解释多少结构。

本章使用轻量的教学特征表，而不是直接训练大型 CNN，是为了让学生先看见形态分类的科学边界：旋臂、边缘视角、浓集度、非对称性、标签投票分歧和观测质量都会共同影响结果。深度模型未来可以加入，但它应该站在 baseline 和错误复查之上，而不是绕过它们。

读这一章时，请把“错分图像”当成核心证据。形态分类最值得学习的部分，往往不是模型给出了哪个标签，而是它在哪些低质量、边界视角或标签不一致的样本上不该过度自信。

\section{案例目标}

这一章把 Part III 里的分类、特征工程、模型评估和可信度判断，放回一个最经典也最容易被误解的天文图像任务：星系形态分类。目标是让读者看到，一个看似“很适合深度学习”的问题，真正做起来其实包含了大量前处理、标签解释和人工复核工作。

\begin{enumerate}
    \item 理解星系形态分类究竟在回答什么科学问题；
    \item 学会从图像 cutout、尺度统一和简单质量控制开始组织数据；
    \item 了解传统手工特征与卷积网络各自适合什么阶段；
    \item 认识类别不平衡、投影效应和标签噪声对分类结果的影响；
    \item 把模型输出重新放回形态学和观测条件中解释。
\end{enumerate}

\section{问题背景：形态分类为什么重要}

星系形态分类并不只是“给图片贴标签”。在天体物理上，它经常和更深的科学问题连在一起，例如：

\begin{itemize}
    \item 星系形成和并合历史；
    \item 恒星形成活跃度与结构之间的关系；
    \item 环境对盘星系、棒旋星系和椭圆星系比例的影响；
    \item 大样本巡天里形态统计和演化趋势的测量。
\end{itemize}

也就是说，形态标签本身往往只是一个中间量。真正的目标，是借助这些标签去问更大的科学问题。

\section{数据：从图像 cutout 到教学特征表}

这一章的正文围绕图像 cutout 工作流展开，但配套 notebook 为了保持轻量可运行，采用的是一个\textbf{从图像中预先提取好的教学特征表}。这意味着我们在本章里先训练三件最重要的能力：

\begin{itemize}
    \item 如何理解形态标签和观测质量信息；
    \item 如何从传统形态学特征建立一个可解释 baseline；
    \item 如何把错分样本重新放回形态学和观测条件中解释。
\end{itemize}

配套 notebook 文件名为 \nolinkurl{ch29_galaxy_}\allowbreak\nolinkurl{morphology_case_}\allowbreak\nolinkurl{study.ipynb}，位于 Part IV notebook 目录中。
对应教学数据文件是 \nolinkurl{galaxy_morphology_case_demo.csv}，位于 \nolinkurl{data/small/}。数据表包含：

\begin{itemize}
    \item 教学用形态标签，例如 \texttt{spiral}、\texttt{elliptical}、\texttt{edge\_on}；
    \item 观测质量与标签可信度信息，例如 \texttt{quality\_flag} 和 \texttt{vote\_fraction}；
    \item 若干传统形态特征，例如 \texttt{axis\_ratio}、\texttt{concentration}、\texttt{asymmetry}、\texttt{smoothness}；
    \item 两个特别适合教学解释的指标：\texttt{spiral\_arm\_strength} 与 \texttt{edge\_on\_score}。
\end{itemize}

真正项目里，数据看上去往往没有想象中整洁。哪怕最终喂给模型的是表格特征，它们背后仍然来自原始图像，因此常见问题并不会消失：

\begin{itemize}
    \item 图像尺度不一致；
    \item 星系中心不在 cutout 正中；
    \item 低信噪比导致旋臂几乎不可见；
    \item 邻近亮源或背景结构污染轮廓；
    \item 人工标签本身存在分歧。
\end{itemize}

所以，形态分类的第一课同样不是选模型，而是先想清楚图像到底可不可比。

\section{第一步：先把图像组织成可比较状态}

对教学案例来说，最基本的前处理通常包括：

\begin{itemize}
    \item 统一 cutout 尺寸；
    \item 保证目标大致居中；
    \item 规范化像素强度范围；
    \item 记录明显的低质量样本。
\end{itemize}

\begin{examplebox}[一个最小图像预处理思路]
\begin{lstlisting}[style=pythonstyle]
image = crop_center(raw_image)
image = rescale_to_fixed_size(image)
image = normalize_pixels(image)
\end{lstlisting}
\end{examplebox}

这一部分看起来“不是 AI”，但它往往直接决定后面的分类结果。一个模型如果主要学到的是背景亮度、噪声模式或 cutout 边缘，而不是星系结构，那么它的高准确率也没有太大意义。

也正因如此，配套 notebook 在进入分类前，先做了两个很朴素但很关键的限制：

\begin{itemize}
    \item 只保留 \texttt{quality\_flag = clean} 的样本；
    \item 只保留 \texttt{vote\_fraction >= 0.70} 的较高一致性标签。
\end{itemize}

这一步的意义，是把“暂时难以解释的图像”和“相对稳定的教学样本”先分开。前者不一定要扔掉，但更适合进入人工复核队列，而不是直接被当作干净监督标签。

\section{第二步：传统特征为什么仍然值得保留}

今天一提到图像分类，很多人第一反应都是 CNN。但在教学和真实科研早期阶段，传统手工特征仍然非常有价值。例如：

\begin{itemize}
    \item 轴比、集中度、非对称度；
    \item 表面亮度分布；
    \item 纹理统计；
    \item 简单的径向轮廓特征。
\end{itemize}

这些特征的好处是：

\begin{itemize}
    \item 解释性强；
    \item 样本较小时更稳；
    \item 有助于判断“图像里到底是哪类结构在起作用”。
\end{itemize}

因此，这一章非常适合把“传统特征 baseline”和“卷积网络 baseline”并列来看，而不是直接跳过前者。图~\ref{fig:ch29_concentration_spiral} 展示了一个最适合先看的二维切面：集中度与旋臂强度。

\section{第三步：投影效应会把盘星系推向边界区域}

星系形态分类最棘手的地方之一，是有些“错分”未必真的是模型错了，而是观测方向本来就在改变可见结构。一个盘星系如果越来越接近 edge-on 朝向，它在图像上就会表现为：

\begin{itemize}
    \item 轴比变小；
    \item 旋臂可见度下降；
    \item edge-on 倾向评分上升；
    \item 与椭圆星系或细长盘星系之间的边界变得更模糊。
\end{itemize}

图~\ref{fig:ch29_axisratio_edgeon} 展示了清洗后样本在“轴比 vs edge-on 倾向评分”空间里的分布，并把 notebook 中最值得复查的两个测试样本标了出来。

\begin{itemize}
    \item 盘星系朝向本身会改变图像可见特征；
    \item 某些椭圆星系在低维特征空间里确实可能靠近边界；
    \item 低 margin、低投票一致度和边界位置常常同时出现。
\end{itemize}

\section{第四步：先把 baseline 做稳，再决定是否上 CNN}

当数据预处理比较稳定、样本量也开始够用时，卷积神经网络会自然展现优势。原因主要有三点：

\begin{itemize}
    \item 它能直接从局部结构中学习旋臂、核、棒和不规则轮廓；
    \item 它不必依赖手工定义每个形态指标；
    \item 迁移学习可以显著降低从头训练的门槛。
\end{itemize}

但要记住：CNN 的优势建立在数据条件也比较靠谱的前提下。如果标签本身噪声很大、低信噪比样本太多、类间边界本来就模糊，那么复杂模型不一定能解决核心问题。对这个案例来说，更合理的路线是：

\begin{enumerate}
    \item 先用教学特征表做一个可解释 baseline；
    \item 再检查哪些样本属于边界、低质量或低投票一致度；
    \item 最后才考虑把原始图像 cutout 接入 CNN 或迁移学习模型。
\end{enumerate}

\begin{examplebox}[本章 notebook 的最小 baseline 流程]
\begin{lstlisting}[style=pythonstyle]
rows = keep_clean_high_vote_samples(rows)
X = standardize(morphology_features(rows))
classifier = nearest_centroid(X_train, y_train)
pred = classifier(X_test)
review_misclassified_samples(pred, rows_test)
\end{lstlisting}
\end{examplebox}

这个 baseline 看上去并不“先进”，但它刚好能回答教材里最重要的问题：\textbf{到底是哪些结构特征在帮助我们分辨样本，而哪些对象需要人工复核。}

\section{第五步：配套 notebook 怎样组织实验}

配套 notebook 采用的是一条非常适合本科教学的小型实验链：

\begin{itemize}
    \item 先统计 \texttt{quality\_flag} 与 \texttt{vote\_fraction}，决定哪些样本进入分析集；
    \item 再比较不同类别在 \texttt{axis\_ratio}、\texttt{concentration}、\texttt{spiral\_arm\_strength}、\texttt{edge\_on\_score} 上的范围；
    \item 然后用标准化后的 8 维传统特征训练最近中心分类器；
    \item 最后回到错分样本，检查最近邻和特征快照。
\end{itemize}

这条链有一个很重要的教学特点：它把“数据筛选、特征解释、模型输出、错误复查”连成了闭环，而不是把 notebook 写成一串孤立 API 调用。即使后面升级到 CNN，也应该尽量保留这种闭环结构。

\section{第六步：类别不平衡、标签噪声和人工复核不能忽视}

在很多真实样本里，最常见的类型通常数量最多，而真正罕见但科学上有趣的类型数量很少。于是你常会同时面对：

\begin{itemize}
    \item 类别不平衡；
    \item 少数类 recall 很差；
    \item 某些类别训练样本过少；
    \item 标签之间的投票分歧。
\end{itemize}

这时仅仅报告 overall accuracy 基本是不够的。你至少还应关心：

\begin{itemize}
    \item 每类召回率；
    \item 哪些类别最容易互相混淆；
    \item 低质量样本是否集中在某些错分区域；
    \item 是否需要把“高不确定度样本”单独交回人工复核。
\end{itemize}

在当前教学数据里，\texttt{vote\_fraction} 的作用尤其直观：它让我们把“标签本身就很稳的样本”和“标注者存在明显分歧的样本”分开。等学生进入更真实的图像任务后，这会自然过渡到：

\begin{itemize}
    \item 对 Galaxy Zoo 投票分布做阈值筛选；
    \item 对边界对象保留“不确定/待复核”标签；
    \item 在模型输出之外加一层人工审阅工作流。
\end{itemize}

\section{为什么这个案例特别适合比较传统方法和现代 AI}

星系形态分类有一个很好的教学特点：它天然允许多条方法线并存。

\begin{itemize}
    \item 传统图像特征 + 经典分类器；
    \item CNN 从像素直接学习；
    \item 迁移学习作为低成本深度学习方案；
    \item 人工复核作为高不确定度样本的闭环。
\end{itemize}

因此，这一章很适合训练学生的工程判断：不是默认“新方法一定更好”，而是结合样本规模、标签质量、算力和科学目标选择恰当复杂度。

\section{一个最小星系形态案例报告模板}

星系形态分类报告必须把图像质量、标签可靠性和模型错误一起写清楚。一个最小报告可以包含：

\begin{enumerate}
    \item \textbf{形态任务定义}：说明类别是 \texttt{spiral}、\texttt{elliptical}、\texttt{edge\_on}，还是更细的形态体系；
    \item \textbf{质量与标签筛选}：报告 \texttt{quality\_flag}、\texttt{vote\_fraction} 阈值，以及被送入人工复核的样本；
    \item \textbf{传统特征 baseline}：说明使用哪些形态特征，以及它们和物理直觉是否一致；
    \item \textbf{分类诊断}：报告混淆矩阵、每类 recall，以及主要错分方向；
    \item \textbf{边界样本复查}：回看投影效应、低 SNR、邻源污染和低投票一致度；
    \item \textbf{CNN 升级理由}：说明深度模型要解决的是局部结构表示问题，而不是替代数据质量控制。
\end{enumerate}

\begin{examplebox}[星系形态案例报告骨架]
\begin{lstlisting}[style=pythonstyle]
morphology_report = {
    "quality_cut": "quality_flag == clean",
    "vote_cut": "vote_fraction >= 0.70",
    "baseline_features": ["axis_ratio", "concentration", "asymmetry"],
    "class_metrics": per_class_metrics,
    "review_queue": ["SP06", "EL05"],
    "cnn_next_step": "compare against baseline only after image QC is stable",
}
\end{lstlisting}
\end{examplebox}

如果报告只写“模型把 SP06 分错了”，读者学不到什么；如果报告写清楚 SP06 的轴比、edge-on 倾向、旋臂可见度和投票一致度，错分就变成了理解形态边界的证据。

这份形态报告也应自然进入 Part IV Model Experiment Packet：质量标记、投票阈值、形态标签和传统特征进入 Dataset Contract；传统特征 baseline、split、分类诊断和 CNN 升级理由进入 Model Experiment Record；混淆矩阵、边界对象和人工复核队列进入 Evaluation artifact 与 Error analysis artifact；标签噪声、投影效应、低质量图像和深度模型适用条件进入 Trust Statement。

\begin{warnbox}[星系形态分类中的常见误区]
\begin{itemize}
    \item 没有处理尺度和居中问题，就直接喂图像给模型；
    \item 用总体准确率掩盖少数类几乎学不到的事实；
    \item 把投影效应导致的边界模糊误当作单纯模型错误；
    \item 忽视人工标签本身的不确定性；
    \item 在样本很小的时候盲目追求从头训练深网络。
\end{itemize}
\end{warnbox}

\section{AI 助手如何帮助你}

在这个案例里，AI 助手很适合帮助你：

\begin{itemize}
    \item 梳理图像前处理和数据增强的实验方案；
    \item 帮你整理混淆矩阵和错分图像复查记录；
    \item 把“传统特征 baseline”和“CNN baseline”写成结构清楚的对照实验说明；
    \item 根据 notebook 输出生成一张待复核对象清单。
\end{itemize}

但到底是标签有问题、图像质量有问题，还是模型真的没学到结构；以及某个高不确定度对象是否值得单独研究，仍然需要你自己判断。

\section{练习}

\begin{exercisebox}[基础练习]
把 notebook 中的标签一致性阈值从 \texttt{vote\_fraction >= 0.70} 改成 \texttt{vote\_fraction >= 0.80}，比较保留下来的样本数、每类样本数以及测试集错分情况。说明你看到的是“更干净的监督集”，还是“过度收缩后的样本”。
\end{exercisebox}

\begin{exercisebox}[进阶练习]
把最近中心分类器替换成 \texttt{3-NN} 或你熟悉的另一个轻量分类器，比较 \texttt{SP06} 和 \texttt{EL05} 的预测是否改变。无论结果是否提升，都要写一段说明：这是否真的意味着更复杂模型更好，还是只是对边界对象做了不同的几何划分。
\end{exercisebox}

\begin{exercisebox}[开放练习]
假设你下一步要把原始图像 cutout 接入 CNN 或迁移学习模型，请写出一个最小实验计划。至少包括：图像尺寸如何统一、训练/验证/测试如何划分、哪些样本应该进入人工复核队列，以及你准备用什么指标证明 CNN 学到的是形态结构而不是观测条件。
\end{exercisebox}

\section{本章小结}

星系形态分类案例提醒我们：图像 AI 任务真正困难的部分，常常不是网络结构本身，而是图像尺度、标签质量、投影效应和科学定义边界。只有把这些问题一起放进工作流里，分类结果才会真正变得可信。对教学来说，先用传统形态特征把 baseline、错分复核和人工审阅逻辑讲清楚，再进入 CNN 和迁移学习，通常比一开始就追逐复杂模型更有价值。
