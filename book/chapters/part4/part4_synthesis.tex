\chapter*{第四部分收束：从经典案例走向深度学习}
\addcontentsline{toc}{chapter}{第四部分收束：从经典案例走向深度学习}

Part IV 的四个案例展示了一个共同事实：很多天文与物理问题并不是一开始就需要深度学习。只要数据规模适中、特征含义清楚、baseline 合理，传统机器学习和清楚的诊断图表就已经能完成大量有价值的工作。

但这些案例也自然暴露出深度学习的入口。星系形态图像中，手工特征很难完全表达旋臂、棒结构和局部纹理；光谱分类中，局部谱线模式可能比全局手工特征更有信息；时间序列和序列化光谱中，局部模式、长程依赖和表示学习会越来越重要。

\section*{什么时候需要更复杂模型}

更复杂模型通常不是因为“更先进”，而是因为简单表示开始吃力：

\begin{itemize}
    \item 图像中有局部结构，手工特征难以覆盖；
    \item 光谱中谱线位置会轻微移动，raw-feature baseline 不够稳；
    \item 样本结构复杂，线性或树模型只能抓到局部规则；
    \item 你希望模型自己学习中间表示，而不是完全依赖人工特征；
    \item 任务需要处理序列、patch、token 或多模态输入。
\end{itemize}

同时，复杂模型也会带来新的责任：更多参数、更高过拟合风险、更难解释、更强算力依赖，以及更严格的数据划分要求。

\section*{四个案例的共同交付物}

读完 Part IV 后，不应只记住“四个 notebook 都跑通了”。更重要的是，你应该能为任意一个案例交出一份最小项目包：

\begin{itemize}
    \item 一个 \textbf{Dataset Contract}，列出关键字段、单位、误差、质量标记、标签来源和选择效应；
    \item 一个 \textbf{Model Experiment Record}，说明科学问题、ML 任务、baseline、模型选择、split、评估和复现入口；
    \item 一个或多个 \textbf{Algorithm Card sidebar links}，说明本案例复用了 Part III 的哪些算法逻辑；
    \item 至少一个 \textbf{Evaluation artifact}，例如 HR 图、残差图、PCA 平面、混淆矩阵或错分样本图；
    \item 一个 \textbf{Error analysis artifact}，说明失败样本是否集中在某类对象、某个物理区域或某种数据质量条件下；
    \item 一个短的 \textbf{Trust Statement}，明确哪些结果可以支持，哪些只是教学数据上的初步现象。
\end{itemize}

如果一个案例只剩下最终分数，却没有这些交付物，那么它仍然只是一次模型调用，而不是一个可复查的小型科学项目。

\section*{一个可以立刻执行的 Part IV 综合任务}

为了确认自己已经能把方法章装配成案例项目，可以任选 Part IV 的一个案例，完成一份最小 case package。这个任务不要求换成更大的真实数据，也不要求使用更复杂模型；它要求你把问题、数据、baseline、模型、诊断图、错误样本和有限结论放进同一个目录中。

\begin{examplebox}[Part IV 最小 case package 目录]
\begin{lstlisting}[style=terminal]
part4_case_package/
  data/
    case_input_demo.csv
  notebooks/
    case_workflow.ipynb
  scripts/
    reproduce_case_result.py
  figures/
    main_diagnostic.png
    error_examples.png
  results/
    dataset_contract.md
    model_experiment_record.md
    trust_statement.md
\end{lstlisting}
\end{examplebox}

建议按下面六步完成：

\begin{enumerate}
    \item \textbf{写清案例问题}：说明它是分类、回归、结构探索还是候选筛选，以及对应的科学目标；
    \item \textbf{完成 Dataset Contract}：列出关键字段、单位、质量筛选、标签来源或图像/光谱处理条件；
    \item \textbf{保留 baseline}：先报告一个简单可解释方法，再说明后续模型是否真正改进；
    \item \textbf{生成诊断图}：至少有一张能支撑科学解释的主图，以及一张错误或残差诊断图；
    \item \textbf{复查失败样本}：列出若干错分、残差大或边界样本，并解释可能原因；
    \item \textbf{写 Trust Statement}：说明当前教学数据支持什么，不支持什么，若升级到真实数据还要补哪些证据。
\end{enumerate}

最终的 \texttt{model\_experiment\_record.md} 和 \texttt{trust\_statement.md} 应该能被同学或助教独立阅读，并回答三个问题：这个案例为什么值得做，模型结果为什么暂时可信，以及哪些边界必须在进入 Part V 或后续项目时保留下来。

\section*{四个案例如何合成一套能力}

这四个案例的价值不在于题目彼此独立，而在于它们共同训练了端到端项目的四种装配能力。Gaia HR 图案例让你从物理图像开始定义任务；光度红移案例让你把连续预测和残差诊断连起来；光谱分类案例让你处理高维序列化特征；星系形态案例则把图像、标签不确定性和现代 AI 入口连到一起。

如果把它们压缩成一句话，Part IV 训练的是：先用最小可信 pipeline 形成案例包，再判断复杂模型是否真的必要。这个顺序会直接决定 Part V 的深度学习是否扎实。

\section*{从案例项目包到后续项目}

Part IV 的案例交付物可以直接迁移到后续更大的课程项目或研究训练中。Dataset Contract 保留数据入口，Model Experiment Record 保留 baseline、模型选择和评估过程，Evaluation artifact 与 Error analysis artifact 保留图表和失败样本，Trust Statement 保留有限结论和公开边界。

这意味着每个案例 notebook 都不只是一次练习。它应该训练学生把数据、模型、图表和文字解释放到同一个证据链中。等进入独立项目时，学生要做的不是重新发明项目结构，而是把这些已经练过的 Model Experiment Packet 扩展到自己的题目。

\section*{进入 Part V 前的基线意识}

Part V 会进入神经网络、卷积、自编码器、Transformer 和现代 AI 工具。但请把 Part IV 的经验带过去：\textbf{深度学习不应该抹掉 baseline，而应该站在 baseline 之上。}

一个健康的深度学习实验，至少应能回答：

\begin{itemize}
    \item 简单模型已经做到什么程度；
    \item 深度模型具体解决了哪个表示问题；
    \item 改进是否来自真正的结构学习，而不是数据泄漏或调参偶然；
    \item 错误样本是否仍然能被人类复查；
    \item 模型输出是否能回到科学问题中解释。
\end{itemize}

如果这些问题答不上来，复杂模型就只是更漂亮的黑箱。
