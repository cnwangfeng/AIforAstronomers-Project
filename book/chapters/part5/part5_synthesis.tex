\chapter*{第五部分收束：把复杂工具带回可信交付}
\addcontentsline{toc}{chapter}{第五部分收束：把复杂工具带回可信交付}

Part V 让我们接触了更复杂的模型和更强的 AI 工具。到这里，一个学生已经可能写出神经网络训练代码、构造卷积特征、解释自编码器重构误差、描述注意力机制，也能用 LLM 协助调试、整理文献和搭建工具调用流程。

但本部分真正要留下的并不是“我会用复杂工具”，而是“我知道复杂工具应该怎样被纳入可信交付”。这两者差别很大。

\section*{复杂度必须有理由}

进入 Capstone 项目前，任何复杂方法都应能回答三个问题：

\begin{enumerate}
    \item 它解决了简单 baseline 的哪个具体不足；
    \item 它引入了哪些新的风险和验证成本；
    \item 它的输出如何回到科学图表、错误样本和报告结论。
\end{enumerate}

如果一个深度模型只是在分数上略有提升，却让数据需求、解释成本和复现难度大幅上升，那么它未必是更好的课程项目选择。相反，一个简单模型如果能被完整解释、稳定复现、清楚展示限制，往往更适合本科 capstone。

\section*{AI 助手也要有交付边界}

LLM 和 agentic workflow 可以显著提高效率，但它们在课程项目中必须留下可审计痕迹：

\begin{itemize}
    \item 哪些代码由 AI 起草，哪些由你验证；
    \item 哪些文献摘要来自 AI 辅助，哪些已经回到原文核对；
    \item 哪些工具调用成功，哪些失败或被人工接管；
    \item 哪些内容需要引用、署名或版权检查；
    \item 最终报告中哪些判断是你自己承担责任的科学解释。
\end{itemize}

这不是为了限制 AI，而是为了让 AI 真正成为科研助手，而不是不可审计的黑箱作者。

\section*{Part V 的可信交付包}

进入 Capstone 之前，Part V 的任何复杂工具都应该能交出一个最小可信交付包。它不要求学生写成正式论文附录，但至少要让别人看得懂：你为什么用了这个工具、它解决了什么问题、它带来了什么风险、你如何检查它。

对深度模型来说，交付包可以写成一个小型 model card：

\begin{enumerate}
    \item \textbf{任务与 baseline}：模型要解决什么任务，简单 baseline 做到什么程度；
    \item \textbf{训练与验证}：训练/验证/测试如何划分，是否保留最终测试集；
    \item \textbf{输入与预处理}：图像、光谱或序列如何标准化，哪些样本进入人工复核；
    \item \textbf{主要结果}：报告指标、错误样本、可视化诊断和与 baseline 的差异；
    \item \textbf{失效边界}：说明样本规模、标签噪声、分布外样本和解释限制。
\end{enumerate}

对 LLM、agent 和 AI 辅助写作来说，交付包应更像一份 usage log：

\begin{enumerate}
    \item \textbf{使用场景}：AI 参与了代码重构、debug、文献整理、报告润色，还是工具调用；
    \item \textbf{材料边界}：哪些材料可以输入 AI，哪些因版权、隐私或未发表状态不能输入；
    \item \textbf{人工核查}：哪些代码已运行，哪些引用已回到原文页码，哪些 claim 已人工签字；
    \item \textbf{失败与接管}：哪些 AI 输出被拒绝、改写或由人类接管；
    \item \textbf{公开声明}：最终报告如何披露 AI 使用，而不是只写一句泛泛的“使用了 AI”。
\end{enumerate}

\begin{examplebox}[Part V 可信交付包骨架]
\begin{lstlisting}[style=pythonstyle]
part5_delivery = {
    "baseline": baseline_summary,
    "model_or_ai_tool": tool_description,
    "validation_evidence": validation_checks,
    "error_or_failure_cases": review_queue,
    "human_signoff": "what the student verified and takes responsibility for",
    "disclosure_note": ai_use_statement_or_model_limit_note,
}
\end{lstlisting}
\end{examplebox}

这份交付包的作用，是把“复杂工具很强”转换成“复杂工具可以被课程项目安全使用”。如果这一包材料写不出来，就说明当前工具还没有准备好进入最终项目结论。

\section*{一个可以立刻执行的 Part V 综合任务}

任选一个深度学习或现代 AI 工具场景，完成一份最小 trusted delivery package。它可以是一个小型神经网络、一个 CNN baseline、一维卷积光谱模型、自编码器异常候选、LLM 辅助编程记录、agent 工具调用记录，或 AI 辅助文献整理记录。重点不是工具多强，而是使用理由、验证证据和人工责任是否闭合。

\begin{examplebox}[Part V 最小 trusted delivery package 目录]
\begin{lstlisting}[style=terminal]
part5_trusted_delivery/
  inputs/
    allowed_materials_note.md
  notebooks/
    tool_workflow.ipynb
  scripts/
    reproduce_or_verify.py
  figures/
    validation_or_error_cases.png
  results/
    trusted_delivery_record.md
    ai_usage_log.md
\end{lstlisting}
\end{examplebox}

建议按下面六步完成：

\begin{enumerate}
    \item \textbf{说明使用理由}：复杂模型或 AI 工具解决了哪个 baseline 或人工流程的具体不足；
    \item \textbf{写清输入边界}：哪些数据、代码、文献或报告内容可以输入工具，哪些不能；
    \item \textbf{保留运行或交互记录}：记录模型设置、随机种子、提示词、工具调用或主要修改；
    \item \textbf{生成验证证据}：包括测试结果、诊断图、错误样本、引用核对或人工复查记录；
    \item \textbf{写出失败与接管}：列出 AI 输出被拒绝、模型失效或人工接管的具体位置；
    \item \textbf{形成披露说明}：说明最终报告中如何描述模型限制或 AI 使用。
\end{enumerate}

最终的 \texttt{trusted\_delivery\_record.md} 至少要回答：为什么使用这个工具，它处理了什么输入，输出如何被验证，哪些结果不能信，哪些部分由学生本人承担责任。若这份记录写不出来，就不要把工具输出放进 capstone 结论。

\section*{可信交付包怎样进入 Part VI}

Part VI 不会要求学生另外写一套孤立的 AI 附录。更好的做法，是把 Part V 的可信交付包嵌入 capstone case template：

\begin{itemize}
    \item 深度模型的任务、baseline、训练验证和失效边界进入 Baseline / Model / Evaluation cards；
    \item 输入预处理、样本质量门、verification loop 和人工复核队列进入 Data / Reproducibility / Evaluation cards；
    \item action log、claim ledger 和 caveat 说明进入 Interpretation / Integrity cards；
    \item LLM、agent 和 AI 辅助写作的 usage log、AI-use statement、human signoff 和公开声明进入 Integrity card 与最终报告。
\end{itemize}

这样处理后，复杂工具不会漂浮在项目之外。它们会被放回同一套证据结构中，和科学问题、数据、baseline、诊断图、错误样本和限制说明一起接受检查。

\section*{进入 Capstone 前的选择原则}

Part VI 会要求学生完成一个完整项目。此时最重要的能力不是会多少模型，而是会做选择：

\begin{itemize}
    \item 能用简单方法回答的问题，不要为了炫技而复杂化；
    \item 需要复杂模型的问题，必须先有简单 baseline；
    \item 需要 AI 助手参与的流程，必须保留验证记录；
    \item 所有最终结论都必须能回到数据、代码、图表和限制说明。
\end{itemize}

换句话说，现代 AI 的正确位置不是替代科学判断，而是帮助你更快、更稳地抵达需要判断的地方。
