本书正文默认读者已经具备最基础的 Python 阅读与修改能力。

如果你还不熟悉变量、容器、条件、循环、函数、模块和轻量面向对象接口，请先完成项目中的 Part 0 先修材料，再进入正文主线。Part 0 采用 notebook-first 组织，目的是补齐必要基础，而不占用本书主体的 AI 与科研工作流篇幅。

可以用下面的清单判断自己是否需要先补 Part 0：

\begin{itemize}
    \item 能否从头到尾运行一个 notebook，并理解单元执行顺序；
    \item 能否修改变量、列表、字典和简单表达式；
    \item 能否读懂 \texttt{Path} 与 \texttt{csv.DictReader} 组成的最小数据读取示例；
    \item 能否读懂 \texttt{if}、\texttt{for}、函数和 \texttt{import}；
    \item 能否理解随机种子、路径和简单输出文件为什么会影响复现；
    \item 能否识别 \texttt{table.read()}、\texttt{model.fit()} 这类对象方法调用。
\end{itemize}

如果这些问题大多已经清楚，可以直接从第 1 章进入正文；如果其中几项还不熟悉，建议先把 Part 0 当作课前热身，而不是把它当作本书正文的一部分。

\section*{本书最后要交出的项目包}

本书不是按“工具列表”来组织的。六个正文部分最终都会汇入一份可以复查、评分、交接和维护的 capstone portfolio。为了从一开始就知道学习目标，可以把最终项目包想成八张互相引用的证据卡：

\begin{enumerate}
    \item \textbf{Project card}：科学问题、范围和可回答性；
    \item \textbf{Data card}：数据来源、字段、单位、误差、许可证和筛选；
    \item \textbf{Baseline card}：简单方法的结果和科学意义；
    \item \textbf{Model card}：模型选择、输入输出、训练验证和失败样本；
    \item \textbf{Evaluation card}：指标、诊断图、误差分析和测试边界；
    \item \textbf{Interpretation card}：主要 claim、局限性、不确定度和 caveat；
    \item \textbf{Reproducibility card}：环境、运行入口、随机种子、Git 记录和复跑说明；
    \item \textbf{Integrity card}：引用核查、AI 使用记录、公开边界和 human signoff。
\end{enumerate}

Part I 会先建立运行、版本、数据入口和图表证据；Part II 把不同类型的天文/物理数据读成可解释样本；Part III--V 把模型、案例和现代 AI 工具放进可验证流程；Part VI 再把这些材料收束成最终项目包。因此，读每一章时都可以问：这一章正在帮助我补哪一张证据卡？
