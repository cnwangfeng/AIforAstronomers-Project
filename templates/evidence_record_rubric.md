# Evidence Record Rubric

用途：Part I/II 统一评分口径。它判断的是：**别人能不能根据记录复查结果？**

---

## Pass

满足以下五点：

1. 输入可定位：能找到原始文件、代码或环境入口。
2. 操作可重跑：知道运行什么命令、脚本或 notebook。
3. 输出可找到：能找到生成的图、表、日志或结果文件。
4. 检查可解释：说明了至少一种质量检查或合理性检查。
5. 限制写清楚：明确说明当前结果不能支持什么结论。

硬条件：

- 路径必须可定位。
- 路径应优先使用相对项目根目录的路径；若使用绝对路径，必须说明项目根目录。
- 脚本、notebook、commit 或 tag 必须在需要时可定位。
- 只写“看起来正常”不算有效检查。

## Revise

有结果，但缺少以下一项或多项：

- 输入路径不清楚；
- 脚本名或运行命令不清楚；
- 输出文件不可定位；
- 输出文件无法对应到具体输入和脚本；
- 缺少版本、commit、tag 或环境信息；
- 检查没有具体依据；
- 限制写得过泛。

## Fail

只给图、截图、口头解释或 notebook，无法判断：

- 数据从哪里来；
- 代码如何运行；
- 图表如何生成；
- 结果是否经过检查；
- 当前结论有什么边界。

---

## Chapter-Specific Requirement

每章最多增加 1 条特殊 Pass 要求。例如：

- Ch01: 写操作前必须有 preview，或说明本章全部为只读操作。
- Ch02: 必须指出哪个 commit 或 tag 生成了哪个输出文件。
- Ch06: 必须说明字段/header、单位、缺失值或质量标记。
- Ch11: 必须说明筛选条件和选择效应。
- Ch14: 不能只报告 best period，必须说明 second-best period 或 alias 风险。
